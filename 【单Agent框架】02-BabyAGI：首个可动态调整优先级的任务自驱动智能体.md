## BabyAGI简介

近年，随着大语言模型的大火，智能体（即AI Agent）重新被人们重视起来。在大语言模型作为智能体的大脑的加持下，人们看见了实现AGI（Artificial General Intelligence，通用人工智能）的希望。智能体可以根据目标自主的对任务进行分解和规划，将其分解为更小、可管理、可执行的子目标，然后逐一达成，从而实现复杂任务的高效处理。

**BabyAGI** 是一个基于[OpenAI](https://zhida.zhihu.com/search?content_id=236888645&content_type=Article&match_order=1&q=OpenAI&zhida_source=entity)，能够**实现任务自驱动** 的**精简智能体框架** 。它能够**根据预定目标自动创建、组织和执行任务** ，并**首次将确定任务优先级** 的能力引入至其框架。在执行任务的过程中，它将使用基于前一个任务的结果的预定目标来创建任务。然后，它基于OpenAI的大模型能力，根据目标创建新任务，并使用Chroma/Weaviate来存储和检索任务结果，以获取上下文。

如果向BabyAGI提出一个目标，系统将不断优先考虑需要实现或完成的任务，以达到这个目标。其主要由如下4个步骤来执行：

  1. 从任务列表拉取待完成任务项。
  2. 将任务发给执行Agent（Execution Agent），执行Agent集成调用OpenAI API和任务上下文完成任务；
  3. 将任务结果进行完善，然后存放到向量DB Chroma或Weaviate中；
  4. 基于前一个任务的执行结果和初始目标，创建新的任务，插入到任务列表中。

## BabyAGI工作原理

BabyAGI采用精简的架构，能够根据预定义的目标自动生成任务、确定任务的优先级和执行任务。BabyAGI主要由任务生成Agent（Task Creation Agent）、执行Agent（Execution Agent）和优先级排序Agent（Prioritization Agent）等模块构成。

图2 BabyAGI的工作原理

### 任务生成Agent

任务生成Agent函数（task_creation_agent()）通过调用OpenAI的API，基于当前目标和前一个任务的结果来创建新任务。该函数包含4个参数：目标、前一个任务的结果、任务描述和当前任务列表。

然后，任务生成Agent函数向OpenAI的API发送一个提示，该API将返回一组新任务的字符串。然后，该函数将以字典列表的形式返回这些新任务，其中每个字典会包含任务的名称。任务列表将会存储在长期记忆（向量数据库）中，供将来参考。

其实现关键代码如下：
    
    
    def task_creation_agent(
            objective: str, result: Dict, task_description: str, task_list: List[str]
    ):
        prompt = f"""
    You are to use the result from an execution agent to create new tasks with the following objective: {objective}.
    The last completed task has the result: \n{result["data"]}
    This result was based on this task description: {task_description}.\n"""
        if task_list:
            prompt += f"These are incomplete tasks: {', '.join(task_list)}\n"
        prompt += "Based on the result, return a list of tasks to be completed in order to meet the objective. "
        if task_list:
            prompt += "These new tasks must not overlap with incomplete tasks. "
        prompt += """
    Return one task per line in your response. The result must be a numbered list in the format:
    #. First task
    #. Second task
    The number of each entry must be followed by a period. If your list is empty, write "There are no tasks to add at this time."
    Unless your list is empty, do not include any headers before your numbered list or follow your numbered list with any other output."""
        print(f'\n*****TASK CREATION AGENT PROMPT****\n{prompt}\n')
        response = openai_call(prompt, max_tokens=2000)
        print(f'\n****TASK CREATION AGENT RESPONSE****\n{response}\n')
        new_tasks = response.split('\n')
        new_tasks_list = []
        for task_string in new_tasks:
            task_parts = task_string.strip().split(".", 1)
            if len(task_parts) == 2:
                task_id = ''.join(s for s in task_parts[0] if s.isnumeric())
                task_name = re.sub(r'[^\w\s_]+', '', task_parts[1]).strip()
                if task_name.strip() and task_id.isnumeric():
                    new_tasks_list.append(task_name)
                # print('New task created: ' + task_name)
        out = [{"task_name": task_name} for task_name in new_tasks_list]
    return out

### 执行Agent

执行Agent是系统的核心部分，它利用OpenAI的API来处理任务。execution_agent()函数接受两个参数：目标和任务，用于向OpenAI的API发送提示。然后，它将以字符串形式返回任务的结果。其主要执行三类任务，web-search、web-scrape和text-completion。执行的结果和收集的信息也将保存在长期记忆中供将来使用。

其实现关键代码如下：
    
    
    # Execute a task based on the objective and five previous tasks
    def execution_agent(objective: str, task: str) -> str:
        """
        Executes a task based on the given objective and previous context.
        Args:
            objective (str): The objective or goal for the AI to perform the task.
            task (str): The task to be executed by the AI.
        Returns:
            str: The response generated by the AI for the given task.
        """
        context = context_agent(query=objective, top_results_num=5)
        # print("\n****RELEVANT CONTEXT****\n")
        # print(context)
        # print('')
        prompt = f'Perform one task based on the following objective: {objective}.\n'
        if context:
            prompt += 'Take into account these previously completed tasks:' + '\n'.join(context)
        prompt += f'\nYour task: {task}\nResponse:'
        return openai_call(prompt, max_tokens=2000)

### 优先级排序Agent

优先级排序Agent将会对任务列表中的任务进行排序并确认其优先级。它通过调用OpenAI的API，并借助OpenAI大模型的推理能力，对任务列表进行重新排序。该函数接受一个参数，即当前任务的ID。然后，它将向OpenAI的API发送提示，并将以编号列表的形式返回重新排序后的新任务列表。其默认是根据order序号进行排序，可以进行扩展。

其实现关键代码如下：
    
    
    def prioritization_agent():
        task_names = tasks_storage.get_task_names()
        bullet_string = '\n'
        prompt = f"""
    You are tasked with prioritizing the following tasks: {bullet_string + bullet_string.join(task_names)}
    Consider the ultimate objective of your team: {OBJECTIVE}.
    Tasks should be sorted from highest to lowest priority, where higher-priority tasks are those that act as pre-requisites or are more essential for meeting the objective.
    Do not remove any tasks. Return the ranked tasks as a numbered list in the format:
    #. First task
    #. Second task
    The entries must be consecutively numbered, starting with 1. The number of each entry must be followed by a period.
    Do not include any headers before your ranked list or follow your list with any other output."""
        print(f'\n****TASK PRIORITIZATION AGENT PROMPT****\n{prompt}\n')
        response = openai_call(prompt, max_tokens=2000)
        print(f'\n****TASK PRIORITIZATION AGENT RESPONSE****\n{response}\n')
        if not response:
            print('Received empty response from priotritization agent. Keeping task list unchanged.')
            return
        new_tasks = response.split("\n") if "\n" in response else [response]
        new_tasks_list = []
        for task_string in new_tasks:
            task_parts = task_string.strip().split(".", 1)
            if len(task_parts) == 2:
                task_id = ''.join(s for s in task_parts[0] if s.isnumeric())
                task_name = re.sub(r'[^\w\s_]+', '', task_parts[1]).strip()
                if task_name.strip():
                    new_tasks_list.append({"task_id": task_id, "task_name": task_name})
        return new_tasks_list

## BabyAGI的优劣势

**优势**

  * 能够处理复杂任务，根据预定义的目标自动生成任务、确定任务的优先级和执行任务。
  * 可以不断从提示和任务结果中学习反馈，以试错的方式做出类似人类的认知决策。



**劣势**

  * 其本身只是一个精简架构，其任务规划完全依赖ChatGPT，复杂任务规划存在缺陷，且工具的集成使用能力很弱。
  * 无法实现跟踪任务的序列化、运行并行任务，不支持实时收集，例如检查电子邮件和文本以获取新任务和信息。



## BabyAGI的扩展实现

### BabyDeerAGI

BabyDeerAGI相较于BabyAGI，有如下优势：

  * 支持并行任务处理。
  * 支持用户通过输入工具输入，并与GPT-3.5 Turbo配合使用。
  * 支持查询改写。



### BabyBeeAGI

BabyBeeAGI是继BabyAGI之后演进的一个高级版本，其旨在在一个任务管理提示中处理多个功能。BabyAGI和BabyBeeAGI之间的关键区别如下：

  * **任务管理Agent**



BabyBeeAGI增加了任务管理Agent，使Agent能够处理更多样化的任务。任务管理Agent负责的具体功能包括：

  1. 跟踪完整的任务列表以及完成/未完成的状态；
  2. 分配任务之间的依赖关系；
  3. 决定何时需要新任务来达到目标；
  4. 指定每个任务要使用的工具；
  5. 以清晰的JSON格式提供结果。


  * **依赖任务**  
BabyBeeAGI中引入了依赖任务的概念，其中一个任务在执行之前依赖另一个任务的完成。引入依赖任务为未来的AI模型中更复杂的任务管理系统奠定了基础。目前，其还不允许并行任务。
  * **适应更短的有限任务**  
原始的BabyAGI框架设计用于无止境的任务，但BabyBeeAGI更适用于更短的有限任务。这一改进使AI能够更有效地处理各种任务，从简单的一次性操作到更复杂的多步骤过程。
  * **工具：网页搜索和网页抓取**  
BabyBeeAGI包括一个框架，用于添加多个工具，特别是网页搜索和网页抓取功能，这些工具使AI模型能够从网络中收集和处理相关信息，进一步增强其完成任务和做出决策的能力。
  * **全局JSON变量和[GPT-4](https://zhida.zhihu.com/search?content_id=236888645&content_type=Article&match_order=1&q=GPT-4&zhida_source=entity)集成**  
BabyBeeAGI删除了向量搜索和嵌入，取而代之的是利用GPT-4架构生成的全局JSON变量。这一改变使AI模型能够更高效地访问和处理信息，从而提高任务完成和决策能力。



###   
**BabyCatAGI**

BabyCatAGI是继BabyBeeAGI之后演进的一个高级版本，他们之间的关键区别如下：

  * **任务创建Agent**  
在BabyCatAGI中，其引入了一次性运行的任务创建Agent，取代了BabyBeeAGI中每个任务之间工作的任务管理器。该变化显著加快了完整任务列表的执行速度。
  * **智能Agent工具**  
BabyCatAGI中引入了“Agent”作为工具，这是一种复杂的工具组合，包括网页搜索，然后循环遍历结果，并抓取每个结果，以提取只与其他任务相关的信息。这个工具可以实现更高效和准确的信息检索。
  * **GPT-4集成和稳定性**



BabyCatAGI依赖于GPT-4架构，以实现更好的性能和稳定性。
