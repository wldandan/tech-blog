# MMG2Skill：网上有海量的教程，Agent 能不能自己把它变成技能

> 互联网上到处都是人类写的操作指南：怎么装软件、怎么做合成表、怎么打牌。它们图文并茂，但写得时候默认读者是人。MMG2Skill 做了一件事：**把网上随手搜来的教程自动拆成 Agent 能执行的 SKILL.md，再用执行轨迹的反馈反复修，直到真的能跑通。** 六个 VLM 底座、三类环境、18 组对照，全部正向。

## 人写的教程，Agent 没法直接吃

网上的人类教程有四个特征：多模态，图片截图配文字。异质，一篇是教程网站，另一篇是 Wiki。噪音多，混着关联任务的步骤和非必要的导航描述。还有一个隐性假设：**读的人是同一个人，在走同一条路径。** 人看到"点击保存按钮"，知道前一步必须已经打开保存对话框。Agent 不知道。

已有的做法要么让人手写技能库，工作量太大、跟不上任务变化；要么让 Agent 靠自己探索，在探索过程中浪费大量试次去重建网上随手就能查到的人类经验。**能不能直接从公开的图文教程里把技能蒸馏出来，让 Agent 边跑边改？**

南京大学和快手联合团队在 **MMG2Skill** 里给出了一个完整的闭环方案。同时建了一个叫 **MMG2Skill-Bench** 的基准，130 个任务，三类环境，评估的**不再是一个 Agent 能不能自己学会，而是它能不能把教程变成技能并且越跑越好。**

## 三类环境

基准跨了三种交互模式。

**MMG2Skill-GUI**：桌面 GUI 操作。Agent 从屏幕观测和 GUI 动作中操作桌面应用，教程是产品文档和图文操作指南。**MMG2Skill-Game**：开放世界游戏（我的世界）。Agent 通过探索、合成和资源采集完成目标，教程是 Wiki 页面和攻略式指南。**MMG2Skill-Strategy**：规则约束下的回合制卡牌（斗地主、麻将）。Agent 根据公开观测和合法动作做回合决策，教程是规则说明和初级策略材料。

三类任务有一个共同条件：**只入选结果可以从 Agent 可见轨迹或公开最终状态中推断出来的任务。** 这叫 success-inferable 边界。分析器只能看到 Agent 自己能看到的轨迹内容，不能触碰隐藏的 benchmark 分数或环境内部状态。这条边界保证了修订过程的信号来源跟真实部署环境一致。

## 闭环：从教程蒸馏，执行，再按轨迹修

MMG2Skill 是一个三阶段闭环，VLM 模型全程冻结不动。

### 第一阶段：教程变成技能

原始教程是一堆 HTML 加图片和截图。**一个 VLM 技能提取器把它标准化成可编辑的 SKILL.md 格式。** 每条技能不是一段自由文本，而是结构化的元组：可复用的操作步骤、适用条件、预期状态线索（操作完后环境应该长什么样）、恢复知识（偏了怎么拉回来）。概念上分成这四层，实现上以带有过程步骤文字、状态描述和引用教程图片的 Markdown 文档来承载。

**这个结构化的价值不在格式，在"可编辑"。** 一坨自由文本的技能，改起来不知道改哪行。拆成条件、步骤、预期状态、恢复策略之后，Revision 的时候可以精确到其中一层。

### 第二阶段：带着技能跑

技能被注入到 Agent 上下文，跟最近 W-1 轮的观测-动作历史持久共存。每个动作都在任务说明、近期历史和当前技能集的联合条件下采样。行动接口保持不变，只是扩展了可用的过程知识。

### 第三阶段：不在 benchmark 上打分，直接看轨迹改

这是 MMG2Skill 最关键的设计选择。

每轮 rollout 结束后，一个分析器只看任务说明和 Agent 可见的轨迹——**它没有 benchmark 分数，没有隐藏环境状态，没有 gold action 序列。** 它输出两件东西：一是轨迹级别的问题诊断 e_k（哪里成功、哪里失败、行为与目标的偏离位置），二是自判结果 r_k（likely_success 表示从可见轨迹来看任务似乎已完成）。

然后精炼器拿原始教程、当前技能、所有累积诊断来更新技能。诊断列表不做截断，过去的诊断一直保留。更新可能加上缺失的检查条件，把模糊的状态线索写清楚，强化已成功的动作模式，或者删掉从教程抄来但实际跑不通的恢复建议。**改完之后只有技能文件的修改被保留下来，Agent 模型不动。**

整个闭环跑最多 N = 5 轮。如果分析器中途判断 likely_success，提前终止，返回当前技能。

## 实验结果

六个 VLM 底座：Claude Opus 4.6、GPT-5.5、Claude Sonnet 4.6、Kimi-K2.6、Gemini 3.1 Pro Preview、Qwen3.6-Plus。

### 全面正向

**18 组模型-环境组合全部正向，没有一组例外。** 跨底座宏平均 gain：+12.8 到 +25.3 个百分点。最大单格提升是 Gemini 在 Game 环境上的 +33.33pp。

增益分布不依赖底座强度。从低起点起跳的模型（Qwen3.6-Plus 在 GUI 上从 15.0% 涨到 40.0%）受益最明显，但强底座同样受益（GPT-5.5 在 Game 上 +6.67pp）。**教程提供的流程知识跟强模型已经内化的过程知识不重叠，教程有自己的信息增量。**

一个典型例子是 Game 环境里的竹子变木棍合成任务。教程明确说竹子可以作为原材料走一套不同的合成路径。不用教程的 Agent 一直在找木头准备走木板合成路线，把整段步骤预算耗在搜索不存在的木材上。用了教程的 Agent 直接从竹子路径走。

### 直接灌教程不可靠

跟 Vanilla 比，**直接把原始教程灌进 Agent 上下文，GUI 上基本持平，Game 上反而跌了 1.67pp。** 人写的教程里混着兄弟步骤、起始状态假设和路径偏离后不再适用的导航文本。Agent 无法区分哪段是当前任务真正的过程知识、哪段是写给旁边子任务看的。

**结构化提取（w/o revision）本身就比直接灌和 Vanilla 都强**，说明把教程从自由文本块拆成可检索的技能条目，是比压缩或检索更安全的起点。但提取只解决了一半问题。**GUI 上提取一次就捕获了大部分收益，桌面操作翻译成命令行风格是直接的。但 Game 和 Strategy 上，修订贡献了超过 90% 的总增益。** 教程写的是"合成木棍"的宏观流程，但背包里的物品状态、合成界面的实际情况、空间交互的具体方式，只能靠真跑几遍填上。Skill.md 的价值不在于它是压缩上下文，而在于它是一个可以写、可以删、可以改的可编辑接口。

### 修订编辑比上下文反馈管用

一个自然替代方案是 Reflexion 风格：把原始教程留在上下文不动，把累积的诊断作为反馈一次次往里贴。但同样五轮预算、同样提前终止策略和同样的分析器下，**Kimi-K2.6 和 Qwen3.6-Plus 在 GUI 和 Game 上仍然不如 MMG2Skill 的 Skill.md 编辑。** 根因反馈最有用的时候，是它被物化为对技能文件的持久修改，而不是作为上下文记忆的临时附加物。

### 修订收益是非单调的

图 4 的跨轮动态说明了另一件事：**跑更多轮不一定更好。** 分析器从模糊证据中推导修正时，修订可能让后续表现走平甚至回调。尤其在 Strategy 环境中，单局牌的方差比 GUI 任务大得多，分析器更容易从单次牌局中提取有偏见的诊断。这催生了第四项实验——能不能用分析器的自判信号做提前终止。

### 提前终止

在成功可推断任务上，用分析器的 likely_success 做停止信号，**在全量跑满 N=5 的对照下，提前终止在任务达成率不降反升的同时，节省了 25% 到 53% 的试次。** 这意味着分析器的自判信号可以作为一个可校准的部署停止条件，不需要 benchmark 分数干预。

## 几个值得追踪的方向

**教程质量对技能质量的影响是未观测变量。** 论文收集了公开的人工教程，但没有系统地控制教程质量。有些写得好，有些全是噪音，有些只覆盖了部分路径。教程质量的下限就是技能质量的下限。在这个框架里，精炼器能修的是"教程跟运行时不匹配"，不是"教程本身的知识就错了"。

**SKILL.md 的四层结构是概念设计，不是强约束。** 论文把技能拆成步骤、适用条件、预期状态和恢复知识四层，但实际实现中这四层塞进了自由的 Markdown 文本里，没有强制字段。修订粒度仍然部分受限于"Markdown 里多少行是一个可修改的独立单位"。更严格的结构化可能会让修订更精细，但也可能增加提取阶段的负担。

**成功推断的边界决定了自进化能走多远。** 目前只在可以从轨迹中推断结果的任务上做闭环修订。在不可推断的任务上，部分信息博弈、涉及隐藏状态的长期任务里，分析器无法判断 likely_success，自进化链条从这一步就开始断裂。

## 最后

MMG2Skill 做了一件事：**把网上的人类教程变成一个 Agent 可以直接改、改了能重跑、跑了能验证的闭环。**

从教程到技能库，中间要补的不仅是代码，还有教程里那些为人写的隐性假设，以及运行时暴露出来的教程跟环境之间的落差。**直接灌教程不可靠。结构化的提取是一个安全的起点。但真正的增益来自提取之后，把每次失败的轨迹转化成对技能文件的一次精确修改。**
