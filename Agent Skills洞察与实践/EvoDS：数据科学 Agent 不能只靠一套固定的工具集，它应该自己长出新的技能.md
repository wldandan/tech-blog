# EvoDS：数据科学 Agent 不能只靠一套固定的工具集，它应该自己长出新的技能

> 现有的数据科学 Agent 把工具当静态原语。数据清洗、特征工程、模型训练、可视化，每个操作都是预设好的固定动作空间里的一个选项。数据科学的探索本身是开放的，反复出现的成功操作模式应该被抽象成可复用的新技能，而不是每次从头发现。EvoDS 让 Agent 在跑任务的过程中**自己合成新技能、验证、存入仓库、下次直接用。** 同时解决了另一个伴生问题：技能和中间结果越积越多，上下文根本装不下。

## 固定工具箱的局限

DS-Agent、AutoKaggle、DeepAnalyze，这些是目前开源数据科学 Agent 中的代表。它们的共同模式是围绕一组预定义的脚本化 pipeline 做编排，工具空间在任务开始前就锁死了。

这跟人类数据分析师的工作方式完全不同。人在反复处理相似的数据模式后，自然会形成可复用的操作习惯。"这类缺失值用中位数填充"，"那个分布看一眼就知道应该用 log 变换"。这些习惯一旦形成，下一次遇到同一个模式时不用再从零推导。而现有的 Agent 每次遇到同款数据清洗步骤，都要重新探索一遍同款代码。

**更麻烦的是，固定工具空间和迭代式探索之间存在一个规模矛盾。** 数据科学的推理链条天然长，从清洗到特征工程到建模到评估，每一轮迭代都会产生大量中间结果和上下文。如果还要把不断膨胀的技能库也塞进上下文，Agent 很快就会撞到推理上下文的上限，触发"lost-in-the-middle"效应。

港科大（广州）的研究团队在 **EvoDS** 里同时处理了这两个问题。做法是把技能获取和上下文管理做成同一个强化学习框架里联合优化的两个目标。不是"先加技能再加压缩"的串行方案，而是让模型在训练中同时学会三件事：怎么完成当前任务、怎么把成功的经验变成可复用的技能、怎么在有限上下文预算里做出最优的保留和丢弃决策。

## 分层多智能体架构

EvoDS 用了一个层级结构拆开数据科学工作流。

**顶层是 Manager Agent。** 它掌握全局视野，做顶层推理、任务分解和跨 Agent 协调。它的动作空间是最小的一层，只保留代码执行工具包，用来看数据、探索环境、跑程序。需要处理具体子任务时，它不自己做，而是委托给下层专门 Agent。

**下层是五个专业化的 Sub-Agent。** Cleaner 管数据清洗，Featurizer 管特征工程，Modeler 管模型训练和调参，Visualizer 管图表和可视化，Debugger 管错误诊断和修复。每个 Sub-Agent 维护自己独立的局部技能空间，Agent 之间的技能空间互不重叠。

这个设计的核心价值在于：**每个专业 Agent 的动作空间只包含它那个领域里的操作，加上它自己根据历史经验培育出来的专属技能。** 一个 Featurizer 不需要在自己的上下文里看到 Visualizer 的技能库，也不需要被 Modeler 的中间产物干扰。任务的全局目标和跨 Agent 协调信息存在共享记忆里，子任务特定的上下文存在各自局部记忆里。

分层直接降低了每个 Agent 的工具选择错误概率。论文的理论分析证明了这一点：当总的工具数量不减少时，把工具分配到互斥的专业空间中，每个 Agent 在自己的空间里做选择的错误率低于一个统一的大工具空间。**优化的最终目标跟信息瓶颈原理对齐：保留对任务关键的信息，过滤无关信号，在受限的上下文预算下做最优信息选择。**

## 自主技能获取：从成功经验里长出新工具

ASA（Autonomous Skill Acquisition）是 EvoDS 让 Agent 自我进化的核心机制。

运作方式是这样的。当一个 Sub-Agent 在执行子任务时尝试了一段从未出现过的成功操作序列，比如一种新的缺失值处理方式、一个从来没用过的模型调参组合，**系统会判定当前技能库无法覆盖这段成功行为。** 如果这段操作符合以下条件：成功完成了子任务、操作模式在已有技能库中没有对应项、不是简单的参数变化而是流程级的新模式，那么就把这段成功轨迹送去技能合成。

合成出来的候选技能不能直接入库。先要验证，在同类子任务上重跑几次，确认成功不是偶然的。验证通过的技能存入该 Sub-Agent 的专属技能仓库，这个名字、功能说明和可执行代码三位一体的技能条目在后续同类任务中可以直接调用，不再需要从零推导。

技能还按使用频度维护了一个淘汰机制。高频被调用的技能保留，长期未被使用的逐渐退场。**这样就避免了一个常见问题：积累的技能越来越好，但大量技能其实从未被复用，只是占着上下文空间。**

## 自适应上下文压缩：在保留什么和丢掉什么之间做决策

上下文爆炸是数据科学 Agent 的必然结果，因为它同时面对着三个膨胀源。迭代式实验产生的长推理历史。中间产物（数据摘要、错误日志、模型输出）的自然累积。以及 ASA 不断扩展的工具集。传统的做法是硬截断，超过某个 token 阈值就丢掉最早的内容。但数据科学推理中，**最早的步骤往往是最关键的前提假设和数据结构快照，丢掉它们比丢掉中间步骤的代价更高。**

EvoDS 的 ACC（Adaptive Context Compression）把压缩做成一个可学习的动作，而不是一个被动触发的事件。

Manager Agent 在每个决策步有两个选择：执行一个实际操作（跑代码、调 Sub-Agent），或者执行一次上下文压缩。压缩动作的具体行为是让模型自己判断当前上下文中的每一段信息是保留、摘要、还是丢弃。模型的压缩决策被纳入强化学习的目标。**压缩错了导致后续步骤失败，对应的压缩决策被惩罚；压缩之后仍能高效完成任务且省了 token，对应的压缩决策被奖励。**

训练信号引导模型自行学会评估每段上下文对当前子任务的信息价值。这跟 ASA 是互补的：ASA 往上下文中加入新的可复用技能，ACC 负责控制上下文总量，保证加进来的技能不会挤掉关键的过程信息。

## 训练：先模仿专家，再靠 RL 自己优化

EvoDS 的训练分两阶段。先 SFT，再 Agentic RL。

**SFT 阶段让一个更强的教师模型在基准任务上跑一遍，收集成功轨迹。** 这些轨迹不光包含环境动作序列，还包含教师模型的 ASA 决策（在哪个时间点合成哪个新技能）和 ACC 决策（在哪个时间点做哪种程度的压缩）。Multi-Agent 架构让 SFT 可以按 Sub-Agent 分轨训练，Manager 的协调轨迹和 Sub-Agent 的专业操作轨迹分开收集、分开微调，互不污染。

RL 阶段把 Agent 放到线上环境里。奖励信号是复合的：任务完成质量（主信号）、技能复用的有效性（辅助信号，鼓励使用已有技能而不是每次都即兴重写）、以及上下文利用效率（压缩动作太激进导致信息丢失会被惩罚）。Manager Agent 和 Sub-Agent 的奖励互相隔离再汇总，多角色联合优化。

**RL 阶段最关键的设计是技能空间和上下文预算同时变化。** ASA 在 RL 过程中还在持续制造新技能，ACC 同时在动态释放空间。Agent 在训练中体验到的就是部署时会遇到的条件，技能持续增长，上下文持续受限。训练不是在静态环境中跑策略优化，而是在动态增长的技能空间中跑策略优化。

## 实验

四个基准：MLAgentBench、DSBench、AutoKaggle、ML-Bench。

**EvoDS 在四个基准上的平均性能比最先进的开源数据科学 Agent 高出 28.9%，同时完全消除了超出 token 上限导致的运行中断。** 所有消融实验的结论都很一致：ASA 和 ACC 各自独立带来正收益，但两者联合使用时增益最大。这说明技能获得和上下文压缩之间确实存在耦合，单独做一项而另一项失控会抵消部分收益。

一个具体的对比：拿掉 ASA 的 EvoDS 变体在长链任务上的表现随着任务步数增长而迅速退化，因为 Agent 在反复重走相似的探索路径而无法从中积累经验。拿掉 ACC 的变体在高迭代任务上一律触发上下文溢出。完整的 EvoDS 在两类任务上都保持了平稳的性能曲线。

## 几个值得注意的问题

**ASA 的质量依赖验证环节的可靠性。** 候选技能需要同类子任务重跑验证，但如果基准中同类子任务太少（某些任务族只有一个实例），验证本身会比较脆弱，可能通过了一个偶然匹配的成功案例但实际上技能没有泛化性。

**ACC 的决策空间是平滑但模糊的。** "摘要到多简"是一个连续选择，不是二元的。当前实现中压缩的具体粒度由模型的自然语言判断能力驱动，没有引入结构化的信息度量，比如每段上下文对最终答案的信息增益估计。如果基座模型本身对信息价值的判断不准，ACC 可能会把关键的中间推理摘要成一句太模糊的概括，导致后续步骤无从参考。

**教师模型决定了 SFT 的上限。** 两阶段训练中，RL 阶段能改进 SFT 阶段形成的策略，但如果教师模型生成的 ASA 和 ACC 决策本身就有系统偏差（比如倾向于生成太保守的技能或太激进的压缩），RL 可能无法在合理的训练预算内纠偏到全局最优。

## 最后

EvoDS 值得关注的地方不在于它在四个基准上跑赢了，而在于它把两个长期被分开讨论的问题放在了同一个优化框架里解决。

**技能获取让你有更多的工具可用，上下文管理让你有空间去用这些工具。** 如果只做技能获取不做压缩，新技能把上下文撑爆，Agent 在长链任务上直接断链。如果只做压缩不做技能获取，上下文是省了，但 Agent 的能力永远是在一个固定空间里循环。EvoDS 的核心判断是：这两件事应该被同一个强化学习目标联合优化，因为它们在部署中是同时发生的。
