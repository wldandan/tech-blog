现在越来越多企业开始部署LLM Agent处理客服、代码开发、数据处理等复杂任务，但几乎所有团队都会遇到同一个瓶颈：Agent上线后效果往往卡在某个固定水平，很难持续提升。如果想要优化效果，就需要收集大量错误样本重新微调模型，不仅要消耗几十甚至上百张GPU的计算资源，还容易出现过拟合，反而影响原有能力的表现。

由伦敦大学学院、香港科技大学（广州）、吉林大学等机构联合开发的**[Memento-Skills](https://zhida.zhihu.com/search?content_id=273103188&content_type=Article&match_order=1&q=Memento-Skills&zhida_source=entity)** 框架，刚好提供了一个成本极低的解决方案。这是一个通用的可持续学习LLM Agent系统，能让Agent自主构建、适配和优化面向特定任务的技能，全程不需要修改底层LLM的任何参数，就能实现能力的持续进化，甚至能自己设计面向新场景的Agent。

## 一、痛点：智能体上线后就面临“冻结”

现代机器学习的核心逻辑，本来就是从经验中不断学习。作为这一领域目前最前沿的成果，大语言模型已经通过小样本学习、有监督微调、后训练等多种技术，在各类场景下表现出了远超传统模型的能力，几乎重塑了整个行业的学习范式。

但讽刺的是，哪怕智能体的学习潜力再强，部署上线的那一刻就被「冻结」了，之后再也不会自主成长。那么，什么是「冻结」的智能体？「冻结」指的是大模型完成预训练和微调后，参数就彻底固定不变了。这类智能体遇到新任务时，只能靠参数里已经编码好的旧知识，还有上下文窗口里能塞下的有限信息来解决问题，根本没法真正从新经验里学习。

图1 大语言模型适配的三种范式

想让冻结的大模型变聪明？传统方法只有一条：重新微调。但这条路的走起来殊为不易：需要成千上万条高质量标注数据，还需要大量的算力成本，结果还不一定就理想，导致现在大多数企业的AI应用上线时是什么水平，用了一两年还是什么水平——不是不想迭代，是迭代成本太高，风险太大，根本迭代不起。

## 二、核心思路：把可执行技能作为经验手册

**Memento-Skills** 的设计完全基于**Stateful Reflective Decision Process（有状态反思决策过程）** 理论，它最核心的突破是把**可执行技能** 作为外部记忆的基本单元，完全区别于传统Agent使用的对话日志、原始交互轨迹等被动记忆。

这种设计的优势非常明显：记忆不再是需要LLM重新理解的原始数据，而是能直接影响Agent执行策略的主动单元。Agent通过不断更新、优化这些技能文件就能提升能力，全程不需要动LLM的参数，也就完全避免了微调带来的成本和过拟合风险。

### 2.1 关键机制：读写反思学习闭环

**Memento-Skills** 的核心运行逻辑是**[Read-Write Reflective Learning](https://zhida.zhihu.com/search?content_id=273103188&content_type=Article&match_order=1&q=Read-Write+Reflective+Learning&zhida_source=entity) （读写反思学习）** 循环，这个闭环完美模拟了人类学习的过程：从经验中学习，犯错后总结教训，提炼成可复用的方法 —— 经验手册，下次遇到类似问题直接使用。

图2 读取–写入反思学习循环

面对新任务，智能体先翻看自己的技能记忆库，挑出最合适的本领（读取）；接着，它调用冻结的大语言模型认真执行（行动）；最后，根据结果反馈反思得失，把新经验存入技能库（写入）。如此循环，越学越强。

### 2.2 基于读写反思式学习的自进化智能体

在Memento 2的理论里，智能体靠读写反思来学习——但它只能笨拙地记下每一步的状态、动作和奖励，像在日记本上不停追加流水账。Memento-Skills把这个想法搬进了真实世界，做了一次关键升级：把被动的“日志本”变成了主动的经验手册 —— “技能库”。

图3 基于读写反思式学习的自进化智能体架构

现在，智能体的“**读取** ”则是检索当前最相关的技能，用它来指导冻结的大语言模型。智能体不再依赖静态提示词或原始历史轨迹，而是靠一个不断主动优化的技能库来行动。这样一来，面对新任务，它总能拿出最趁手的本领，实现真正的策略改进。

而“**写入** ”不再是单纯记录，而是像一位工匠，根据执行结果直接修改可复用的技能工件 —— 代码、提示词、规范文件。它评估自己的表现，然后重写这些技能，让策略在文件夹里真正被优化。

  * 如果执行成功，提升对应技能的效用评分，增加其后续被选中的概率。
  * 如果执行失败，先进行**失败归因** ：定位到具体哪个技能的哪个环节出了问题，然后对其进行针对性优化，比如添加边界条件、修改执行逻辑、调整Prompt模板。
  * 如果现有技能完全无法覆盖当前任务场景，自动生成全新的技能，经过测试后加入技能库。



  


图6 Memento-Skills的组件架构

Memento-Skills一大妙处在于会把聊天记录、任务轨迹这些经验提炼成可复用、可执行的技能，且每次技能更新都会经过**自动单元测试关卡** 的验证：系统会自动生成相关测试用例，验证修改后的技能在历史任务上的表现，确保修改不会越来越差，不会影响原有能力的效果。

### 2.3 [技能路由器](https://zhida.zhihu.com/search?content_id=273103188&content_type=Article&match_order=1&q=%E6%8A%80%E8%83%BD%E8%B7%AF%E7%94%B1%E5%99%A8&zhida_source=entity)：解决传统技能检索的致命缺陷

传统的技能检索方案（比如BM25、语义向量检索）有个非常致命的先天缺陷：它们完全依赖文本语义相似度做匹配，很容易出现「看起来语义相近，但实际执行逻辑完全不搭」的错误。比如用户问「怎么申请退款」和「怎么重置密码」，这两个句子的语义相似度很高，传统检索器很容易把它们归到一类，但实际上两者对应的处理流程完全不一样，调错技能就会导致整个任务失败。

Memento-Skills 的技能路由器用了一套全新的训练思路，从根源上解决了这个问题：

  1. **优化目标从「语义匹配」变成「任务成功」** ：基于单步离线强化学习训练，优化目标是**执行成功率** 而不是语义相似度，让路由器学会判断“用这个技能能不能完成任务”而不是“这个技能的描述和用户问题像不像”。
  2. **硬负例对比学习，精准区分相似技能** ：训练时采用多正例InfoNCE损失做对比学习，会专门生成大量「硬负例」——就是那些和目标技能同领域、术语相近但实际根本不适用的查询。通过这种方式「刻意刁难」模型，让路由器学会精准区分看起来很像但实际完全不同的技能。
  3. **混合检索架构，兼顾召回率和准确率** ：先用稀疏BM25召回和稠密向量检索分别得到候选结果，再用分数感知的倒数秩融合方法合并两路结果，最后还可以选择加交叉编码器做重排，进一步提升最终的匹配准确率。



## 四、实验验证：效果提升显著，技能自动进化

研究团队用两个行业通用的权威基准测试了Memento-Skills的效果，覆盖不同类型的任务场景：

**[GAIA](https://zhida.zhihu.com/search?content_id=273103188&content_type=Article&match_order=1&q=GAIA&zhida_source=entity)**

GAIA包含165个真实世界的复杂问题，需要多步推理、多模态处理、工具调用等综合能力。团队把数据集分成100个训练样本和65个测试样本：

  * 训练集成功率从第一轮的65.1%提升到第三轮的**91.6** %，三轮迭代后几乎所有训练任务都能正确处理。
  * 测试集准确率达到**66.0** %，比没有技能优化的baseline高出13.7个百分点。
  * 训练后技能库从初始的5个原子技能**增长到41个** ，覆盖了GAIA所需的大部分工具调用和推理能力。



GAIA的测试也揭示了一个特点：因为GAIA的问题非常多样化，不同问题之间的技能复用率较低，所以测试集的提升幅度比[HLE](https://zhida.zhihu.com/search?content_id=273103188&content_type=Article&match_order=1&q=HLE&zhida_source=entity)小，这也说明技能迁移的效果和领域结构的相关性很高。

**HLE**

HLE包含2500个专家级学术问题，覆盖数学、人文、自然科学、工程等8个学科，非常适合测试跨任务的技能迁移能力。团队选择了788个训练样本和342个测试样本：

  * 训练集成功率从初始的30.8%提升到第三轮的**54.5%** ，所有学科的表现都有明显提升。
  * 测试集准确率达到**38.7%** ，是baseline（17.9%）的两倍多，证明在结构化的学科领域，技能的迁移效果非常显著。
  * 不同学科的提升幅度差异明显：人文学科的准确率最高达到66.7%，生物学60.7%，化学62.4%，而工程学科的提升幅度最小，最终准确率42.1%，这也和工程问题的多样性更高、技能复用难度更大有关。
  * 训练后技能库从初始的5个原子技能增长到235个，自动聚集成8个语义连贯的领域簇，包括搜索/网页（48个）、量子/物理（47个）、数学/化学（44个）、代码/文本（38个）、下载/验证（28个）、临床/Excel（27个）、象棋/游戏（20个）、Python/脚本（19个），完全覆盖了HLE的所有学科领域。



实验还验证了理论上的收敛性：随着技能库的增长，记忆覆盖半径会逐渐减小，性能提升的边际效益递减，和Memento 2的理论预测完全一致。当技能库覆盖了大部分常见任务场景后，继续增加技能带来的提升就会变小，系统进入稳定收敛状态。

## 五、总结与应用展望

**Memento-Skills** 的最大意义在于打破了“持续学习必须更新模型参数”的固有认知，证明不断增长、自我优化的技能库可以作为独立的非参数智能层，任何冻结的LLM都可以基于这个层实现能力的持续进化。

这种架构的优势非常适合生产场景：

  * 成本极低：不需要昂贵的微调资源，只需要普通的服务器就能运行。
  * 可控性强：每个技能都是可解释、可修改的，出了问题可以直接定位到具体的技能文件，不像黑盒模型那样难以排查。
  * 收敛有保证：有完整的理论支撑，系统的性能提升路径是可预测的。
  * 兼容性好：可以对接任何LLM，不需要对模型做任何修改。



未来这种自进化Agent的应用场景非常广泛：

  * 客服Agent：可以不断从用户反馈中优化回答和处理流程，不需要每次都微调模型。
  * 代码开发Agent：可以不断积累不同场景的代码生成技能，团队使用越多，生成准确率越高。
  * 科研Agent：可以不断积累不同领域的文献阅读、实验设计、数据分析技能，成为科研人员的得力助手。
