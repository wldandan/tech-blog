# 一、核心解决三大落地死穴

SkillOpt本质上是把深度学习的训练思路直接搬到了智能体技能优化上，核心逻辑非常好懂：**把纯文本的技能文档当成可训练的外部状态，用类似模型训练的流程去优化它，全程不用碰模型权重**，完美命中了当前智能体优化的三个死穴：

1. **解决手改提示词的玄学问题**：之前改提示词全靠经验，改多了容易出现规则冲突，改了 A 坏了 B，完全没法预期效果，SkillOpt 把它变成了可控的迭代流程，每一步修改都有验证，绝对不会越改越差；
2. **解决微调成本过高的问题**：不用动模型权重，优化成本只有微调的几十分之一甚至百分之一，小团队也玩得起；
3. **解决优化结果不可复用的问题**：优化出来的技能是纯文本，和模型、执行环境解耦，能跨模型、跨环境迁移，不用重复优化。

# 二、核心设计：四个机制把优化从玄学变成可控流程

SkillOpt的设计思路非常巧妙，完全参考了深度学习训练的成熟逻辑，但是用在了纯文本优化上，四个核心机制完美解决了不稳定、难复用的问题：

![alt text](image-1.png)

## 2.1 有界编辑预算（Textual Learning Rate）：每次最多改 3-4 条规则，小步快跑不翻车

这个对应深度学习里的「学习率」概念，说白了就是每次迭代最多改 N 条规则，别一次改十几条，很容易规则互相冲突，把之前的好效果全冲没了。

论文里默认是每次最多改 4 条，用余弦调度，一开始可以改多一点，后面迭代慢慢改少一点，完全和互联网产品小步迭代的思路一致：每次改几个点，验证没问题再继续，不会上来就整个重构，直接崩掉。

这个机制直接解决了手改提示词最容易出现的问题：一次改十几条规则，互相冲突，效果雪崩，找都找不到哪里出问题。

## 2.2 验证门禁（Held-out Validation Gate）：效果不涨直接扔掉修改，绝对不越改越差

这是 SkillOpt 效果稳定的核心法宝：每次修改出的新版本技能，必须在**完全独立的验证集**上跑一遍，**只有效果严格超过上一版本，才会接受这个新版本，就算持平都不更新，否则直接扔掉这次修改**，完全杜绝负优化。

这个就和产品上线前的灰度验证是一个逻辑：效果不好绝对不正式上线，永远不会出现越改越差的情况。

## 2.3 拒绝编辑缓冲区（Rejected-edit Buffer）：踩过的坑绝对不踩第二次

所有被验证门禁打回的无效修改，都会被记录到这个缓冲区里，后面迭代的时候，优化器会先看之前哪些修改是没用的，不会再提出同样的无效修改，加快优化收敛速度，避免重复踩坑。

这就相当于你们项目组的 bug 记录，之前改了这个规则导致效果下降，后面就不会再改同样的东西了。

## 2.4 跨纪元慢更新（Epoch-wise Slow/Meta Update）：沉淀长期有效规则，不会被小修改覆盖

每迭代 2-3 轮之后，SkillOpt 会做一次慢更新：把多轮验证都有效的核心规则，写到技能文档的一个受保护的区域，这些核心规则不会被后面的迭代小修改覆盖，相当于把核心功能沉淀下来，不会被后面的小需求改掉。

比如你做表格处理智能体，多轮验证下来**必须先检查表格结构再处理**这条规则是绝对有效的，就会把它写到保护区域，后面迭代不会把这条规则改掉，保证核心能力不会退化。


# 三、完整落地流程：六步跑通，普通团队也能复用

SkillOpt 的整个流程非常清晰，完全可以复用到你自己的业务场景里，不需要复杂的基础设施，手动跑都能跑通：

### 第一步：准备基础物料

- **初始技能文档**：不用写得很完美，把你现在正在用的系统提示词整理一下就行，把基本规则说清楚就够用；
- **数据集**：用你业务里的真实 case 就行，按照**2:1:7**的比例分成三类：训练集（跑轨迹用，20%）、验证集（验证修改效果用，10%）、测试集（最终评估效果用，70%），三个数据集绝对不能有重叠，否则会出现过拟合；
- **自动评估脚本**：能自动给智能体的执行结果打分，不需要 100% 准确，只要能区分好坏就行，实在没有自动评估脚本，小批量人工打分也能跑。

### 第二步：批量跑轨迹收集数据

用当前版本的技能，在训练集上批量跑任务，收集完整的执行轨迹，包括：输入是什么，调用了什么工具，返回了什么结果，最终输出是什么，得分是多少，失败的原因是什么。

不用太复杂，把成功和失败的案例分分类就行。

### 第三步：分析轨迹生成修改建议

把成功和失败的轨迹分开，用一个能力较强的大模型作为优化器（比如 GPT-4o、Claude 3.5 Sonnet 都可以），分析失败轨迹的共性问题，生成具体的修改建议，每次最多生成有界编辑预算规定的条数，比如 4 条，多了也不要。

重点：**所有修改建议必须是通用规则，不能是针对某个具体 case 的硬编码**，比如不能写「遇到用户 ID 是 123 的直接返回 xx」，要写「遇到用户 ID 格式不符合规则的直接报错引导修正」。

### 第四步：新版本验证门禁验证

把修改后的新版本技能，在独立的验证集上跑一遍，计算得分：
- 如果得分**严格高于**上一版本，就接受这个新版本，作为下一轮迭代的基础；
- 如果得分没涨甚至降了，就直接扔掉这次修改，把这次修改记录到拒绝编辑缓冲区，后面别再提同样的修改。

### 第五步：多轮迭代直到收敛

重复第二步到第四步，迭代 3-4 轮基本就能收敛，效果不会再明显提升。

每迭代 2-3 轮，可以做一次跨纪元慢更新，把已经验证非常有效的核心规则写到技能的保护区域，避免被后面的小修改覆盖。

### 第六步：输出最终技能

最终输出的技能是一个纯文本的 md 文件，一般只有几百到 2000 token，普通人就能读懂，也可以直接人工修改，部署的时候直接插到系统提示词里就行，**零额外推理成本**。


# 四、效果有多猛：52个场景全SOTA，技能可跨环境复用

论文里的实验结果可以说是碾压级的，在 6 个不同类型的基准测试、7 个不同大小的模型、3 种不同的执行环境下的 52 个测试场景里，SkillOpt 全部拿到了最优或者并列最优的结果：

![alt text](image-2.png)

1. **提升幅度大**：在 GPT-5.5 直接聊天场景下，**平均提升 23.5 个百分点**，表格处理、办公文档处理等强规则场景最多提升 39%；小模型提升更大，GPT-5.4-nano 平均提升 26.7 个百分点，有些场景直接性能翻倍。
2. **跨执行环境可用**：在 Codex 执行环境下优化的表格处理技能，直接放到 Claude Code 环境下用，**直接提升 59.7 个百分点**，基本不用重新优化。
3. **跨模型可用**：在 GPT-5.4 上优化的技能，直接用到更小的 GPT-5.4-mini 和 GPT-5.4-nano 上，依然能拿到 80% 以上的提升效果。

更有意思的是，优化出来的技能里的规则，都是人类能看懂的非常合理的规则，比如：
- 表格处理场景的规则：**先检查工作簿结构和公式，把计算后的静态值写到整个目标范围，不要依赖 Excel 的自动重算**；
- 办公文档 QA 场景的规则：**把解析好的页面作为主要证据，锁定表格、日期、单位上下文，只输出要求的四舍五入后的值，不要带多余的标签**；
- 数学题场景的规则：**在最强陈述的选择题里，按照定理强度排序选项，优先选择有论证的更强结果选项，而不是选择正确但更弱的推论**。

这些规则和人类专家做了很多次业务之后总结出来的经验完全一致，完全可读、可审计、可修改，完全不用担心黑盒问题。


# 五、小团队零成本落地建议

SkillOpt 的落地门槛非常低，普通团队不需要什么复杂的基础设施就能用，给大家几个实际的落地建议：

1. **先小范围验证，不用一开始就搞全自动化**：先拿 100 个标注过的业务 case，手动跑流程试试，跑 2 轮看看有没有效果，效果好再考虑写脚本自动化，不要上来就搞复杂的框架，浪费时间；
2. **优化器模型不用和部署模型一致**：训练的时候用能力强的大模型当优化器（比如 GPT-4o），优化出来的技能可以用在小模型上，依然能拿到不错的提升，这样训练效果好，部署成本低；
3. **不用纠结参数，默认参数就够用**：论文里的默认参数鲁棒性非常强，编辑预算 4、迭代 3-4 轮，直接用就能拿到不错的效果，不需要花大量时间调参；
4. **最终的技能文档一定要人工审核一遍**：虽然自动优化出来的规则大多合理，但是还是要过一遍，把不符合业务逻辑的规则改掉，避免踩坑。
