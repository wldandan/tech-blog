# TASER：模型接连学 19 个完全不同类型的任务，怎么不把前面的忘光

> 持续学习的标准设定是任务之间多少有点关联。但如果今天数学推理、明天代码生成、后天情感分析呢？

## 一个更残酷的场景

持续学习研究怎么让模型一个接一个学新任务不忘旧的。EWC 用弹性权重巩固保护关键参数，经验回放存一小批旧样本反复练，O-LoRA 把不同任务分配到正交的低秩子空间互不干扰。这些方法在各自的设定下效果都挺好。

但它们都默认任务之间有点关联。要么共享相似的语义，要么输入输出格式一致，要么能力维度接近。实际部署不太可能这么友好。一个 Agent 今天数学推理，明天法律摘要，后天创意文案——这些任务在语义、目标、推理模式、格式上差别极大。

华东师大、复旦和中国海洋大学的联合团队在 TASER 里直接把任务异质性拉到了极端。他们搭了一个叫 HeteroCLBench 的基准：**19 个任务横跨 9 个认知维度**，语言理解、数学逻辑推理、知识事实性、代码生成、多语言、安全伦理、对话交互、情感智能、规划决策全塞进去了。从 SQuAD 问答到代码理解，从跨语言翻译到数学推理，从毒性检测到食谱生成——任务之间几乎没有语义重叠。

在这种设定下，持续学习的挑战从"别忘"变成了"怎么在不同任务之间既隔离专用能力，又复用共享能力"。

## TASER：把容量拆成原子技能，按需分配

TASER 的思路很直：**把模型的可学习容量组织成一个动态增长的原子技能库，每个 LoRA 模块的一个 rank 维度就是一个原子技能。** 新任务来了，先探测它和已有技能的差异多大，再决定加多少个新原子技能。多了浪费容量还干扰旧的，少了学不会。

三个组件。

Atomic Skill Incremental Learning 是 TASER 最核心的东西。每次来新任务，不是固定分一个 rank=8 的 LoRA，而是动态决定要几个新原子技能。

做法分三步。第一步，搞一个过度参数化的探针 LoRA（probe adapter，最高 rank 可以到 8/16/32），在新任务数据上训几步。不需要收敛，只需要产生一个足够稳定的奇异值谱，能反映任务的更新方向偏好。

第二步，把探针矩阵投影到已有技能空间的正交补上。已有技能的下投影矩阵拼一起，QR 分解得正交基，探针减去在历史子空间上的投影，剩下的残差 A_diff 就是新任务需要、但已有技能覆盖不到的方向。

第三步，对残差做 SVD，选最小秩 r_new 使保留的奇异值能量占比超过 θ_energy（默认 0.9）。r_new 就是分配给这个任务的新原子技能数。任务越陌生，残差能量越大，分的技能越多。越接近已有技能，残差小，靠复用就够了。

结果很直观。SQuAD 问答和跨语言翻译这类生成性强、结构性高的任务分到高 rank（能到 7-8），MathQA 和毒性检测这类分类/推理任务用一两个原子技能就够了。分配模式在 r_max=8、16、32 三种预算下高度一致，SVD 信号反映的是任务本身的表征需求，不是超参调出来的假象。

Orthogonality-Enhanced Skill Detection 管的是技能隔离。训练新 LoRA 时加两项正交约束。L_inter 惩罚新技能和所有历史技能下投影矩阵的相关性，保证技能之间能隔离。L_intra 惩罚新技能内部各 rank 维度之间的相关性，防止秩坍缩。分配了高 rank 的陌生任务上，多个新原子技能必须填不同方向，否则分再多也是冗余。

Skill Dynamic Routing 用轻量两级 MLP 门控路由器（带 bottleneck），每个输入产生 Sigmoid 权重向量，选择性激活相关原子技能。用 Sigmoid 不 Softmax，因为异构任务可能同时需要多个技能。加 L1 稀疏惩罚，鼓励每个输入只激活最少技能。

消融数据很说明问题。完整 TASER 在 HeteroCLBench 上 AP 63.31、BWT -0.94。去掉 Atomic Skill Incremental Learning 改为固定 rank=8，AP 掉到 57.14（-6.17），BWT 掉到 -2.69。去掉 Skill Dynamic Routing，AP 崩到 44.82（-18.49），BWT 崩到 -5.39——没有动态路由，异构任务之间的干扰直接碾碎了已有技能。去掉正交约束也有明显损失。

## 数据

AP 63.31，离多任务联合训练的 upper bound（65.70）只差 2.39 个点，BWT 接近零（-0.94）。Seq FT 的 BWT 是 -13.54，CL-MoE 是 -15.13。防遗忘最接近的是 I-LoRA（-1.56），但 AP 比 TASER 低了不少。

换 Qwen-2.5 做 backbone，AP 64.65（MTL upper bound 64.95），BWT -1.33。放到更传统的 LNT 基准（15 个 NLP 任务）上，AP 86.68，BWT 仅 -0.05——几乎没有遗忘。

## 两个做得好的地方

SVD 能量阈值的敏感性分析扎实。r_max=8 的紧凑预算下，AP 和 BWT 在多个 θ_energy 下都很稳。r_max=32 的大预算下如果 θ_energy 调太低、保留了太多奇异值方向，性能反而崩——过多的"新技能容量"引入的方向和旧技能产生了干扰。TASER 不是靠堆容量来提分的，是容量分配机制在压冗余。

动态 rank 分配的尺度不变性。同一组任务在 r_max=8、16、32 三种预算下的 rank 分配模式高度一致，生成型任务始终要更多 rank，分类型始终要更少。SVD 驱动的分配看起来像"任务复杂度的稳定估计器"，不是调参包装。

## 局限

每个任务只分一个 LoRA 模块，通过 rank 调技能数。极度异构的任务，把多方向技能塞进一个模块是不是最优？也许同一任务该允许分多个模块。

探针阶段额外算力开销论文没细算。虽然不用训到收敛，但流式场景下多跑一个探针始终是附加成本。

技能库只加不减。每轮加，19 个任务攒 19 个 LoRA 模块。任务流无限延长迟早碰容量天花板。论文没讨论技能合并或淘汰策略。
