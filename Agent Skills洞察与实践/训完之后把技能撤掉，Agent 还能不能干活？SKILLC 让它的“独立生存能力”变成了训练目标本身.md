# 训完之后把技能撤掉，Agent 还能不能干活？SKILLC 让它的"独立生存能力"变成了训练目标本身

技能增强 RL 已经是一个成熟的范式了。训练时给 Agent 装技能让它学得更快，推理时继续带着技能执行。这没问题。

但有一个更终极的问题：**如果把技能当成训练期的"辅助轮"，训完之后撤掉，Agent 还能不能自己骑？**

这就是技能内化（Skill Internalization）。目前的代表性方法是 SKILL0：训练中逐步减少技能注入，控制课程节奏，到训练末期 Agent 已经在没有技能的条件下跑了大部分 rollout。

**SKILL0 有一个盲区：它知道技能还有没有用（用来控制什么时候撤技能），但它在优化的时候不区分"这次成功是靠技能"还是"这次成功是靠自己"。所有 rollout 混在一起归一化，梯度看不见这两种成功之间的差别。** 美团的新工作 SKILLC 就是来补这个盲区的。核心设计叫**对比技能信用分配（CSCA）**：在同一个策略更新里同时采样"有技能"和"没技能"两组 rollout，直接对比差异，然后把差异转化成优化信号。

在 ALFWorld 上，SKILLC 超过 SKILL0 5.5 个百分点。在 WebShop 上超 4.4 个百分点。**而且这都是在推理时不使用任何技能的情况下做到的。**

---

## 一、内化盲区：为什么现在的技能内化方法不够

先理清三种范式。

**技能增强 RL（Skill-Augmented）。** 训练时用技能，推理时也用技能。SKILLRL、D2SKILL 都是这条路。技能是终身伴侣。

**技能内化 RL（Skill-Internalization）。** 训练时逐步撤掉技能，推理时不带技能但期待 Agent 已经学会了自己完成任务。SKILL0 走这条路。

SKILL0 的做法是用一个课程门控来控制技能注入的强度：早期大部分 rollout 带技能，晚期大部分不带。门控的决策依据是"技能对当前任务类型还有没有用"。如果验证集上有技能和没技能的成功率已经差不多了，就说明内化完成了，可以撤技能了。

**这是好的课程控制，但不是好的优化。** 因为在每一次 GRPO 更新里，所有 rollout（不管是带技能的还是不带技能的）都混在一起做优势归一化。当带技能的 rollout 占多数时，归一化抬高了整个批次的奖励基线。不带技能的 rollout 即使靠自己成功了，在归一化后的优势也被人为压低了。优化器看不到"这个 Agent 已经能独立完成任务了"，它只看到"这个批次总体表现不错"。

**SKILLC 管这个叫"内化盲区"（Internalization Blindness）。优化目标里没有区分"靠技能成功"和"靠自己成功"，Agent 就永远得不到"独立成功值得鼓励"的信号。**

## 二、SKILLC 的三个机制

### 2.1 配对的对比 rollout：同时跑"有技能"和"没技能"

SKILLC 的第一个动作很简单：**对每个活跃任务类型，在同一个策略更新里同时采样两组 rollout。一组带技能注入，一组不带。两组来自同一个策略参数。**

这个设计跟 RESKILL 的 within-group skill testing 是同一种思路，利用 GRPO 天然的分组结构做受控对比。不一样的是，RESKILL 对比的是新版本技能 vs 旧版本技能，SKILLC 对比的是有技能 vs 没技能。

从两组 rollout 的奖励均值差异中，提取任务级的对比信号 $\hat{\Delta}(x)$。**这个信号不是用来给轨迹加分的，它纯粹用于信用分配和课程控制，不会被转换成奖励或技能加分。**

### 2.2 双流优势估计：让梯度"看到"独立成功的价值

这是 SKILLC 最关键的设计。

标准 GRPO 把两类 rollout 混在一起归一化。SKILLC 把优势分解成两个互补的流：

**流一：全局质量流。** 保持标准 GRPO 的全局归一化，维持现有策略优化的稳定性。

**流二：对比修正流。** 基于任务级对比信号 $\hat{\Delta}(x)$，对不带技能的 rollout 施加单向修正。**只修正不带技能的 rollout 向上，不修正带技能的 rollout。** 意思是：如果你没用技能也成功了，你的优势应该被额外抬高，因为这是"独立成功"，比"靠技能成功"更值得鼓励。

修正强度由内部化进度动态控制。任务类型在训练中越接近内化完成、两种条件下的成功率差距越小，修正幅度就越大。

**这个单边修正的设计非常精准。它没有改变全局排序，最强的 rollout 仍然是最强的。但它让"不带技能也成功了"的 rollout 在优化中获得更高的权重，从而使策略参数朝着"不靠技能也能行"的方向更快移动。**

### 2.3 验证驱动的自适应课程

双流优势估计需要知道"当前这个任务类型的内化进度怎么样"来决定修正强度。这个信息来自验证集。

每隔 $d$ 步，策略在验证集上用"有技能"和"没技能"两种条件各跑一次，测量两种条件下的成功率差异。这个差异经过 EMA 平滑和 Sigmoid 激活后，驱动三个课程参数的联合控制：

**归属强度 $\lambda_{\text{eff}}$。** 内化差距越大（技能还有用），修正力度越小（当前阶段的重点是先学会，不是先独立）。差距越小（快要内化完成了），修正力度越大，逼 Agent 扔掉拐杖。

**技能注入比例 $\rho$。** 内化进度越大，带技能的 rollout 占比越小。从早期的大部分带技能渐进到晚期只保留少量探针预算，用于持续监测内化状态。

**活跃技能集 $K_{\text{active}}$。** 当一个任务类型的验证对比信号持续稳定为零（有技能和没技能的成功率长期一致），说明技能对这个任务类型已经没有额外帮助了。该类型从活跃集中移除，不再产生带技能的 rollout，也不再参与对比修正。单调收缩，不会恢复。

**三个课程参数全部由同一个验证对比信号驱动，不需要人工调参。**

## 三、实验说了什么

ALFWorld（具身家居，六个任务类型）和 WebShop（网络购物），用 Qwen3-4B 做基座，对比 SKILL0 等技能内化基线。

### 无技能推理下超 SKILL0 5.5 个百分点

**ALFWorld 上 SKILLC 达到 83.1%，超过 SKILL0（77.6%）5.5 个百分点。WebShop 上超过 SKILL0 4.4 个百分点。** 两个都是推理时完全不使用技能。

**跟技能增强方法（推理时带技能）相比，SKILLC 同样有竞争力。** 在 WebShop 上 SKILLC 的无技能推理分数超过了 SkillRL 的带技能推理分数。

### 信号质量决定效果边界

论文做了一个关键的特征分析：**CSCA 的效果高度依赖于对比信号的质量。**

在 Heat 和 Cool 这两个冷却和加热子任务上，CSCA 带来了 +10.5% 和 +11.1% 的大幅提升。因为这两个任务类型的"有技能 vs 没技能"对比信号非常清晰。Pick2 上甚至 +33.4%。

但在 Pick 子任务上，CSCA 反而让她退步了 11.5%。**原因是 Pick 任务本身比较简单，种子基线已经很高，可用的对比信号太弱，两种条件下的成功率几乎没有稳定差异。在这种情况下，CSCA 的修正流反而引入了噪声。**

在只有一个技能类型的环境（WebShop）中，增益也较小（+3.1%）。对比机制在有多个不同技能类型、每个有不同内化进度的环境下发挥最大价值。

### 内化不需要牺牲性能

**一个重要的结论：SKILLC 训练出来的策略，在无技能条件下的表现，逼近甚至超过了 SKILL0 训练出来的策略在有技能条件下的表现。** 训练结束时，Agent 已经不再需要技能来维持高性能。它真的学会了独立。

## 四、这篇论文在说什么

SKILLC 回答了一个很本质的问题：**技能是训练工具还是终身伴侣？**

如果是终身伴侣，那技能增强 RL 就够了，训完接着用。如果只是训练工具，理想情况下训完可以扔掉让 Agent 独立运作，那就需要内化机制。SKILLC 把内化从"课程控制问题"升级成了"信用分配问题"：不是简单地决定什么时候撤技能，而是**在每一次优化中让 Agent 知道："不带技能也能成功"比"靠技能成功"更值得学。**

这个想法跟 RESKILL 是在同一个方向上，都是把"技能不应该是静态的外部资源"这个认知往前推了一步。RESKILL 让技能和策略共同进化。SKILLC 更进一步：**技能不光要一起进化，进化到最后还应该被策略吸收、内化、可以被扔掉。**
