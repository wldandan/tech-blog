# SIRI：推理时不需要技能库，因为技能已经长进模型里了

> 现有的技能型 Agent 方案把技能当作推理时的持久依赖：训练时用外部模型生成技能，推理时从技能库里检索、塞进 prompt。SIRI 走了条不同的路。**技能只在训练时出现，训练结束时消失。推理时的模型就是原始 prompt，没有技能库，没有检索服务，没有任何外部记忆。** 技能已经被选择性内化进模型参数里了。

## 技能库是一个推理时依赖

技能对于长交互 Agent 的价值是公认的。它们把成功的经验压缩成高层行为抽象——"目标对象可见且可达时，先拾取它再继续探索"、"网购时先用硬约束筛选再做偏好比较"。这些抽象降低了探索负担，提升了学习效率。

但现有方案对技能的使用方式有共同的结构：**技能是外挂的。** 训练时依赖外部模型或被分开维护的技能构建模块来从轨迹中生成技能。推理时依赖从技能库中检索相关技能注入 prompt。

这个结构带来了三个连锁成本。训练端，轨迹到技能的转换需要额外的模型调用或专门维护的技能管理器，工程复杂度和训练开销同步上升。推理端，每次决策都要先检索、再注入上下文，加长了 prompt、增加了延迟、增加了部署复杂度。最要命的一点是：**agent 的改进部分来自辅助模块，而不是来自策略学习本身。** 技能库一旦加载错误、检索漂移或维护滞后，推理质量直接被连带影响。

厦门大学、美团和澳门理工的联合团队在 **SIRI** 里问了一个更根本的问题：**能不能反过来，把技能只当作训练时的临时信号？** 训练中用，训练完就扔。推理时模型回到原始 prompt，但已经把技能里最有用的部分吸进了参数里。

## 三阶段课程：热身、挖矿、内化

SIRI 是一个三阶段框架。每一阶段解决一个不同的问题。

### 热身：先会跑，再谈技能

**直接从没训过的策略中挖技能是危险的。** 低质量的轨迹里夹杂着偶然成功和无效行为，从中蒸馏出的技能可能把噪音伪装成启发式，在后续训练中被放大。

Phase 0 先用 GiGPO 做一轮基础策略训练。GiGPO 提供了两层优势信号：轨迹级 episode-level advantage 区别整条路是否优于平均，锚点状态级 step-level advantage 区分同一状态下不同动作的效用。合成优势信号驱动策略优化，不做技能增强。这个阶段产出的不是策略，是高质量的成功轨迹缓存，为后续阶段提供可靠的行为证据。

### 自挖矿：策略本人就是技能生成器

传统做法里，技能是外部模型读轨迹写出来的。SIRI 换过来：**当前策略本人阅读自己的成功轨迹，从中总结出紧凑的技能描述。**

这听起来像是让策略分身同时做两件事，既做执行者也做反思者。但论文做了两个关键设计来防止自我总结退化为自我合理化。

第一，**候选技能必须通过在线效用验证。** 策略写出一组技能 m 之后，用同一个任务跑两组 rollout：一组注入技能 m 做技能增强推理，一组不加任何技能做基线推理。两组出来后直接比效果。只有技能增强推理在任务成功率上显著优于基线推理时，这组技能才被"晋升"为有效技能进入技能库。否则丢弃。这样做的效果是把技能发现中的主观成分替换为可测量的在线信号。"我觉得这个总结对了"变成了"这个总结确实提高了成功率"。

第二，**技能被组织为结构化键值对。** 每条技能包含一个场景描述（什么时候适用）和一条策略文本（怎么行动）。检索时用上下文和技能的语义相似度匹配。这种结构让技能成为可定位、可独立验证的单元。验证失败时知道是哪条具体技能有问题，而不是整个技能库的问题。

**自挖矿模块输出的技能质量，能在训练后期追上闭源大模型蒸馏出来的技能。** 论文做了一个对照实验：把自挖矿的技能生成器替换为更强的外部 LLM，观察训练全程的性能差距。差距持续收敛，说明随着在线验证和内化的推进，自生成的技能变得越来越好用。

### 选择性内化：不是所有技能词都值得学

技能增强推理确实提高了任务成功率，但这不意味着技能引导的每个 token 都值得蒸馏进纯策略。**直接模仿所有技能增强响应可能学进无关的语言模式而非有用的动作模式。**

SIRI 的选择性内化机制在两层上做了过滤。

**轨迹层。** 用 GiGPO 的 trajectory-level 处理效应判断整条技能增强轨迹是否比同一任务的纯策略基线轨迹更有价值。只从处理效应为正（技能确实带来提升）的轨迹中取样内化。

**动作层。** 用 GiGPO 的 step-level 优势信号挑选具体哪些动作 token 值得吸收。动作 token 的优势越高，表示在当前状态下这个动作的选择越优于基线水平。**只有来自技能增强轨迹且动作优势为正的 token 才参与内化损失的计算。** 来自技能增强轨迹但优势为负的 token、来自基线轨迹的任何 token、以及任何非动作相关的文本 token，全都不参与。

内化损失的权重还额外引入了一个**技能衰减因子 α**，控制技能在每个 token 上的贡献比例。α 在训练过程中逐渐下降。早期技能对模型是全新的外源信息，偏差大，需要更高的权重推动学习。后期技能越来越接近模型自身的策略分布，模型应该逐渐依赖自己的推理而非技能拷贝。

所有内化训练结束后，**推理时模型回到原始 prompt，没有任何技能库、检索服务或外部记忆。** 技能结束了它的生命周期。

## 实验

ALFWorld（具身家务）和 WebShop（网购），一个底座：Qwen2.5-7B-Instruct。

**ALFWorld 上，SIRI 把 GiGPO 从 0.908 拉到 0.930。** 对比 SkillRL，SIRI 从 0.899 提升到 0.930。

**WebShop 上，SIRI 把 GiGPO 成功率从 0.728 拉到 0.813，score 从 0.844 拉到 0.899。** 对比 SkillRL，成功率从 0.727 提升到 0.813。

对比范围覆盖了四类基线：prompt 型（ReAct）、标准 RL（GRPO、Tree-GRPO、GiGPO）、记忆增强型（Reflexion、Expel、Agent-Q）和技能增强型（SkillRL、Skill-SD）。**SIRI 在所有对比中取得最优整体表现。**

消融实验说明了几件事。

**去掉自挖矿验证环节，让技能不经验证直接入库用于训练**，WebShop 成功率显著下降。没有在线效用筛选的噪声技能污染了内化过程。

**去掉优势加权的动作层过滤，直接内化所有技能增强响应**，性能同样下降。再次说明不是所有技能引导的 token 都该被学进去。

**用外部强 LLM 替换自挖矿的技能生成器**，训练早期的性能差距明显，晚期差距收敛。自挖矿通过在线验证和内化追平了外部强模型的技能质量优势。

## 三个值得注意的点

**GiGPO 的两层优势是 SIRI 关键基础设施。** SIRI 的轨迹层过滤和动作层选择都依赖 GiGPO 生产的优势信号。如果底层 RL 信度不够，优势估计噪声太大，那么两层过滤的精确性都会受损。SIRI 的有效性某种程度上绑定在 GiGPO 的质量上。

**技能衰减因子 α 目前是启发式设计的。** 论文用了递减的 α 来中和技能在训练后期的偏差，但没有做系统性的 α 策略消融。如果技能质量在训练过程中不是单调改善的，某些轮产出了低质量技能，递减 α 可能会让好技能的信号也衰减得太快。

**内化之后的策略透明度是一个开放问题。** SIRI 训练完，技能被吸收进参数就不再可查。如果推理时出了错，没办法像技能增强型方案那样直接翻"当时注入了哪条策略"来找原因。透明度和可解释性上的损失是内化路径的固有代价。

## 最后

SIRI 把这个方向的问题从"怎么更好地生成和管理技能库"拧成了"怎么让技能库变成训练过程中的一次性脚手架"。

建楼的时候搭脚手架，楼盖好之后拆掉。SIRI 的思路就是在训练结束时拆掉技能库的脚手架，只留下模型自己的能力。**技能是训练时的梯子，不是推理时的轮椅。**
