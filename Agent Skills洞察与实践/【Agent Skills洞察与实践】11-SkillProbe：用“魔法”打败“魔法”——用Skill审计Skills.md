## 一、研究背景与核心挑战

随着LLM Agent生态的快速扩张，以[ClawHub](https://zhida.zhihu.com/search?content_id=271996446&content_type=Article&match_order=1&q=ClawHub&zhida_source=entity)为代表的中心化技能市场成为Agent能力增强的核心基础设施。技能作为封装的流程化知识单元，其语义决策与执行层解耦的架构，带来了传统安全方案无法应对的两大核心安全挑战：

  1. **语义-行为不一致** ：技能的自然语言文档与底层可执行代码逻辑存在偏差，恶意开发者可在代码中植入文档未声明的影子功能、越权操作，而Agent仅基于语义层文档进行调用，无法感知执行层的恶意行为。
  2. **跨技能组合风险** ：单独审计无风险的技能，在多技能协同调用的执行链中，可通过上游不可信输出触发下游高风险行为，形成emergent协同攻击；组合爆炸问题也让传统原子级审计完全失效。



**现有防御方案存在显著局限** ：以智能体为中心的运行时治理，聚焦实时干预，多采用提示词层面防御，但无法消除隐藏在技能逻辑深处的隐秘漏洞；静态代码分析仅聚焦离散的代码级漏洞，无法覆盖语义层偏差与跨技能组合式威胁。

## 二、SkillProbe框架核心设计

SkillProbe是一个多阶段、模块化的多Agent协同安全审计框架，核心采用“**Skills-for-Skills** ”设计范式，**将每个审计环节封装为标准化技能模块** ，让Agent通过加载这些审计技能，完成对目标技能的全流程安全校验，实现审计逻辑与框架核心引擎解耦，具备极强的扩展性。

### 2.1 整体架构

框架分为五层核心架构：

  * **输入层** ：兼容本地目录、压缩包、远程URL等异构格式的审计目标接入与预处理；
  * **编排层** ：以安全审计Agent为核心调度中枢，维护审计状态、分发任务，驱动多阶段流水线执行；
  * **技能层** ：框架核心，包含三大审计模块（[Gatekeeper](https://zhida.zhihu.com/search?content_id=271996446&content_type=Article&match_order=1&q=Gatekeeper&zhida_source=entity)、Alignment Detector、Flow Simulator），对应全流程审计能力；
  * **输出层** ：将审计结果合成为结构化安全报告与多维度安全评分卡；
  * **基础设施层** ：提供工具注册表、审计指纹持久化数据库、Agent运行时环境等底层能力。



框架采用“以技能审计技能” 范式，将每个审计阶段抽象为标准化技能包，每个包封装自然语言规范（定义审计意图）、机器可读工作流配置（程序化实现）、系统提示词与操作脚本。

同时提供两种运行模式：快速模式（串行执行准入过滤与一致性检测，用于高频初筛）、标准模式（全阶段并行执行，用于深度系统级安全评估）。

### 2.2 三阶段核心审计流水线

**阶段1：Gatekeeper（准入过滤）**

采用“解耦检查+集中聚合”架构，将现有安全工具封装为独立执行器并行检测，覆盖合规性、恶意代码模式、风险依赖、权限声明合理性等维度。遵循保守原则，任一检查项不通过即全局拦截，仅全维度通过的技能可进入下一阶段。

**阶段2：Alignment Detector（语义-行为一致性检测）**

将一致性问题形式化为文档声明能力集D(s)与代码实现能力集C(s)的匹配问题，通过文档提取器与代码提取器并行解析，经语义归一化后，构建四类对齐矩阵实现风险分类：

  * Match：完全对齐；
  * Over-declaration：过度声明权限；
  * Under-declaration：存在影子功能（高风险）；
  * Mixed：复杂偏差。



**阶段3：Flow Simulator（组合风险模拟）**

通过风险指纹标签化，将组合风险检测的复杂度从指数级O(2^N)降至线性级O(N×Rules)，解决组合爆炸问题。

首先，系统会给每个技能打上两类标签：输出风险标签（描述技能输出的风险特征）、输入敏感标签（描述技能输入的风险接收特征）。

随后，基于定义的9类核心风险链路策略（覆盖命令注入、间接提示词注入、数据泄露、意图劫持等典型攻击模式），系统会批量执行攻击路径模拟，校验技能之间是否会形成危险的攻击链。

为了避免误报，系统还引入了证据强制机制：Agent识别的每一个漏洞，都必须引用原始代码或文档里的对应片段作为证据，确保审计结果的可追溯、可验证。

### 2.3 审计裁决机制

基于三大维度（恶意模式、语义一致性、组合安全性）构建安全评分卡，采用一票否决原则，将技能分为三个安全等级：REJECTED（存在关键漏洞）、CONDITIONAL（低风险偏差需人工复核）、APPROVED（全维度符合安全基线）。

## 三、实验评估与核心发现

研究基于ClawHub平台的真实技能数据集，结合8个主流LLM系列（含高推理编码专用模型、轻量化模型）开展大规模评估，核心发现如下：

**审计的不可能三角：严格度、细粒度、低延迟不可兼得**

**高审计严格性、高风险发现粒度、低延迟三者难以同时实现** 。其中[Claude Sonnet 4.6](https://zhida.zhihu.com/search?content_id=271996446&content_type=Article&match_order=1&q=Claude+Sonnet+4.6&zhida_source=entity)是最严格的审计模型（45%的CONDITIONAL率），[Gemini 3.1 Flash-Lite](https://zhida.zhihu.com/search?content_id=271996446&content_type=Article&match_order=1&q=Gemini+3.1+Flash-Lite&zhida_source=entity)延迟最低（单技能平均18.5s），[GPT-5.2-Codex](https://zhida.zhihu.com/search?content_id=271996446&content_type=Article&match_order=1&q=GPT-5.2-Codex&zhida_source=entity)可提供最多的修复建议，SkillProbe在不同模型下均保持了高风险识别的鲁棒性。

**人气 - 安全悖论：下载量从来不是安全的代名词**

对下载量Top2500的技能审计发现，仅9.9%的技能处于完全无风险状态，59.8%的技能通过基线审计但存在轻微安全缺陷，30.4%的技能因安全违规被标记为CONDITIONAL；**超90%的高流行度技能未通过严格安全审计** ，且技能风险率在不同下载量层级中保持稳定，下载量完全无法作为安全质量的可靠指标。

**全连通的风险网络：级联攻击已是系统性风险**

499个存在明确风险链路的技能，形成了包含75373条风险边的高连通性风险关联网络，其中数据泄露（32328条边）、事实投毒（28441条边）占总风险边的80%。

**所有高风险技能在风险维度形成单一巨型连通分量** ，意味着级联风险是系统性而非孤立的，单一模块的漏洞可通过技能链产生multiplicative攻击面放大效应，传统原子审计完全无法覆盖此类风险。

## 四、给Agent生态的三大安全启示

SkillProbe的研究，不仅提出了一套可落地的审计方案，更给整个高速发展的Agent生态，带来了三个至关重要的安全启示。

  1. **技能市场必须建立标准化的前置审计体系，摒弃 “盲信人气” 的错误认知** 。当前ClawHub等平台的安全扫描，仍停留在发布后的特征码匹配阶段，无法覆盖语义欺骗和组合风险。未来的技能市场，必须把多维度的安全审计，作为技能发布的强制准入门槛，彻底消除用户对下载量的盲目信任。
  2. **安全审计的重心，必须从纯代码扫描，升级到语义 - 执行的对齐校验** 。Agent时代，技能的自然语言描述层，已经成为核心攻击面。传统的代码审计，只能发现已知的漏洞特征，却无法防御 “文档合规、代码恶意” 的语义欺骗。未来的安全体系，必须把语义一致性审计，放到和代码审计同等重要的位置。
  3. **防御范式必须从原子化的组件检查，升级到组合化的全链路模拟** 。当高风险技能已经形成全连通的风险网络，单技能的审计已经无法应对系统性的级联攻击。未来的Agent安全架构，必须具备对多技能执行链的风险预判能力，在技能被调用前，就完成组合风险的模拟与拦截。



## 五、总结

LLM Agent的终极愿景，是让AI成为能自主完成复杂任务的智能体，而技能市场，就是这个智能体的 “能力器官”。但如果我们不能给这些器官建立严格的安全检疫机制，再强大的能力，也只会变成黑客手里的武器。

SkillProbe的出现，不仅填补了Agent技能市场的安全治理空白，更给整个行业敲响了警钟：在Agent能力高速进化的同时，安全基建必须同步跟上。只有构建起可信任、可扩展的安全治理体系，Agentic的愿景才能真正安全落地。
