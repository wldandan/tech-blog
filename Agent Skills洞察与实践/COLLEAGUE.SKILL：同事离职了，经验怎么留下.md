# COLLEAGUE.SKILL：同事离职了，经验怎么留下

> 资深同事离职，脑子里的判断标准、审核规范、事故处理直觉跟着一起走了。COLLEAGUE.SKILL 做的事就是把这种事工程化——把人留下的零散痕迹蒸馏成一套可安装、可修正、可版本管理的 Agent 技能包。

## 不是还原一个人，是打包一套经验

LLM Agent 现在能装技能了。Claude Code 有 skill 机制，AgentSkills 标准定义了 SKILL.md 格式，越来越多 Agent 宿主开始支持加载外部技能包。工具让 Agent 能做事，技能让 Agent 知道怎么做。

但技能从哪来？手写的说明书太费人，trajectory 蒸馏出来的任务模式只管得了自己。两类方案都漏掉了一类知识源：**人。**

一个工程师的代码审核标准不会写在某份文档里。它散落在 PR 评论、事故复盘、聊天决策、设计文档的批注里。一个公共知识分子的推理风格横跨几十篇长文、访谈、演讲。这些不是写好的指令，是埋在异质痕迹里的隐性经验。

上海人工智能实验室的 **COLLEAGUE.SKILL** 把这件事定义成了一个形式化问题：**给定一组关于某个人的痕迹材料，蒸馏出一个结构化技能包，里面包含可检查的能力规则、交互风格约束、来源边界和使用限制。** 输出不是一个藏在 prompt 里的隐式 persona，而是一组文件，可以打开看、可以修正、可以回滚、可以装到不同的 Agent 宿主上。

GitHub 上目前约 18.5k star，公开画廊 215 个技能、165 位贡献者，累积画廊 star 超过 10 万。

## 双轨制：把"做事"和"说话"拆开

COLLEAGUE.SKILL 把一个技能包劈成了两条独立的轨道。

**能力轨**管的是做什么、怎么做：工作流程、技术标准、审核准则、决策启发式、从过往工作中提炼的教训。在代码审核技能里，这条轨写的是"先检查认证逻辑、输入验证、速率限制、响应 schema、敏感数据暴露，这些优先级高于格式问题"。

**行为轨**管的是怎么表达、怎么交互：沟通风格、互动姿态、边界约束，外加一份修正记录。这条轨的职责比常见 persona 系统窄得多。它不模拟人的完整个性，只存一组可检查的行为约束和交互偏好。

两条轨道各自有独立可调用的入口点。**使用时可以加载完整技能，也可以只加载能力轨，只学做事方式、不管表达风格。或者只加载行为轨，只参考沟通模式。** 这个设计压在一根线上：做 persona 系统最容易崩的地方，就是把事实知识、过程性判断和表面语气三种东西搅在一起。拆成独立文件之后，每一层都可检查。

## 从痕迹到技能包，五步

**痕迹摄入。** 系统接受飞书、钉钉、Slack、微信 SQLite 导出、邮件存档、PDF、截图、Markdown、直接粘贴文本。收集器和解析器把异质格式统一转成本地知识目录。

**预设路由。** 目前有三个预设。同事预设（colleague）面向工作场景，证据来自企业内部工作痕迹，强调工作实践和审核判断，治理规则涉及组织访问权限和交接流程。公众人物预设（celebrity）面向公开领域，优先用第一人称作品和长篇访谈，要求标注引用来源和证据边界，禁止私有信息推断。关系预设（relationship）面向私密人际场景，材料是用户自己提供的私人交互记录，本地所有权、可删除性和非公开默认是最高原则。

三个预设不重复造管线。**它们共享同一套蒸馏核心，只在来源范围、证据要求、同意假设和治理规则上做配置级区分。** 加一个新预设只需要改 prompt 配置，不用动代码。

**双轨蒸馏。** 这是管线的核心。能力轨从材料中提取持久的工作方法、专家启发式或来源可追溯的心智模型。行为轨提取表达模式、交互规则和风格边界。两条轨用不同的 prompt 策略，但共享同一份痕迹材料。

**产物写入。** 共享的 writer 组件按 schema v3 渲染所有输出文件。产物清单：SKILL.md（组合后可调用的技能）、work.md 和 persona.md（可编辑的源文档）、work_skill.md 和 persona_skill.md（独立可调用的子技能）、manifest.json 和 meta.json（安装、画廊和生命周期元数据）。

**分发。** 产物可以检查、安装到各种 Agent 宿主（Claude Code、OpenClaw、Codex、Hermes），或选择性地提交到公开画廊。

## 修正是一等公民

生成的技能包不是一次性产物。**修正流程被设计成了生命周期的一等操作。**

修正入口是自然语言反馈。用户可以直说"他不会这么说"、"她在这里会追问"、"这个判断不对，应该看日志而不是看数据库"。能力方面的修正，系统生成 Markdown patch，匹配二级标题定位到具体章节后替换，未匹配的追加到末尾。行为方面的修正，系统生成标准化记录：{scene, wrong, correct}。

每次修正触发版本管理。writer 先归档当前版本，应用 patch 或修正记录，递增生命周期版本号，重新生成所有派生产物。**版本管理器可以列出版本历史、备份当前状态、回滚到任一历史版本、清理旧存档。** 用户可以放心改，改错了随时退回去。

## 公众人物技能：当证据来源是公开信息

公众人物预设是同事预设之外的一个自然延伸。拿一个公共知识分子的公开材料——长文、访谈演讲、公开的决策记录——生成一个思维风格技能，目的不是模仿 Ta 聊天，而是把 Ta 处理问题的方式变成一组可调用的心智模型和推理偏好。

公众人物扩展做了两件同事预设不做的事。**第一，内置一套研究型工具链**：字幕下载、音频转录、字幕清理、研究笔记合并，以及多维度质量检查——心智模型覆盖率、局限性声明、表达模式、内在张力、来源链接、版权安全信号。**第二，强制记录证据边界**：哪里证据充足、哪里推理不完整，显式标注在产物里。证据薄弱的区域不填通用 persona 文本填充，而是主动声明"这里证据不足"。

效果是技能的使用者能清楚看到哪些指导来自哪些公开材料，哪些判断因为证据不足应该谨慎对待。用来源可追溯替代模糊印象。

## 关系预设：最敏感的一层

关系预设把同一条蒸馏管线搬到了私密人际领域。它的定位不是"让 Agent 代替一个人"，而是让私人交互痕迹被表达为本地可编辑、可删除的结构化状态。

相比同事和公众人物预设，关系预设对同意、保留、访问控制和选择性共享有更强的假设。**删除不是"未来某天要做的功能"，而是产物合同中的一等操作。** 本地所有权和非公开默认是架构需求，不是部署策略。

论文把这层列在扩展能力而非推荐用例里。同一套产物格式在治理需求最强的场景里也能运作，但未经约束的使用存在情感过度依赖、非同意模拟和私聊数据滥用的风险。

## 产物不等于人

论文在这一点上划得很清楚。

**COLLEAGUE.SKILL 不声称生成的技能是对一个人的忠实模型。** 它声称的是：选定的痕迹可以转化为一个技能标准兼容的产物，包含显式文件、元数据、入口点、修正记录和生命周期操作。产物的价值不在"像不像"，在于部署之前就能检查它的结构、来源边界、更新行为和部署兼容性。即使不做任何下游人因研究或任务绩效评估，仅从产物本身就判断得了它的架构质量。

这条线把 COLLEAGUE.SKILL 从行为克隆的讨论里抽了出来。它不是在做一个更像人的模型，而是在做一个更可检查的包装格式。

## 最后

COLLEAGUE.SKILL 把经验留存这件事工程化了：**从零散痕迹到结构化技能包，从一次性产物到可修正可回滚的生命周期，从单个 Agent 的内部记忆到跨宿主可安装的开放格式。**

18.5k star 和 215 个公开技能说明了一件事：很多地方有这个需求。同事离职、专家退休、合作结束，这些场景里的知识损失一直在发生，缺的不是重要性的共识，是一套低摩擦的结构化方案。COLLEAGUE.SKILL 填的就是这个缺口。
