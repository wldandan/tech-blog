来自香港科技大学、清华大学、浙江工业大学的《SkillReducer: Optimizing LLM Agent Skills for Token Efficiency》论文，聚焦[LLM编码](https://zhida.zhihu.com/search?content_id=272523468&content_type=Article&match_order=1&q=LLM%E7%BC%96%E7%A0%81&zhida_source=entity)Agent技能的Token效率痛点，通过大规模实证研究揭示了技能膨胀的系统性问题，提出了两阶段优化框架**SkillReducer** ，在大幅缩减Token消耗的同时，甚至提升了技能的功能表现。

## 一、研究背景与核心问题

近年来，以Claude Code、Cursor、[Windsurf](https://zhida.zhihu.com/search?content_id=272523468&content_type=Article&match_order=1&q=Windsurf&zhida_source=entity)为代表的LLM编码代理正在重塑软件开发工作流，而技能（Skill）作为预打包的领域知识与指令集合，已成为扩展代理能力边界的核心载体。Skill的设计初衷是复用知识、节省token。但当前技能普遍存在**技能膨胀（Skill Bloat）** 问题，反而推高了token成本、造成LLM注意力稀释，且缺乏传统代码成熟的优化体系。

为量化该问题，研究者们对**55315个公开技能** 开展了大规模实证研究，发现三大系统性低效问题：

  1. **路由层** ：描述要么没有，要么太长。26.4%的技能完全缺失路由描述，44.1%的技能描述缺失或不足20 Token，其他部分则堆砌无关内容，这些信息对区分技能毫无价值。
  2. **技能体层** ：技能主体中仅38.5%是可执行的核心规则，超60%为背景说明、示例、模板等非可执行内容，却会被全量注入上下文窗口。
  3. **文件层** ：14.8%的技能附带参考文件，单次调用可能加载数万无关Token。



这种普遍存在的技能膨胀现象带来了显著的经济代价：每次技能调用成本0.03-0.15美元，大团队每月花费数百美元。更糟糕的是，过多的无关内容会稀释大模型的注意力资源，增加幻觉产生的概率，最终影响用户体验。

## 二、核心方法：SkillReducer两阶段优化框架

受软件去膨胀技术的启发，SkillReducer框架针对技能的双层结构特性设计了针对性的优化流程，核心目标是在满足路由等价性和功能保留性两大约束的前提下，最大限度降低技能的Token消耗。

图1 SKILLREDUCER概览。阶段1，路由层优化；阶段2技能主体重构。

### 2.1 阶段1：路由层优化

针对描述缺失/描述冗余问题，实现最小充分的路由描述。该阶段采用”**快速压缩 + 实体验证** “的两阶段设计，在保证压缩效率的同时避免过度精简导致的路由错误。

**1、模拟预言的[Delta调试](https://zhida.zhihu.com/search?content_id=272523468&content_type=Article&match_order=1&q=Delta%E8%B0%83%E8%AF%95&zhida_source=entity)**

在快速压缩阶段，团队设计了模拟路由Oracle，用于评估技能描述的路由正确性。Oracle接收测试查询集与候选技能池（包含目标技能、四个通过TF-IDF余弦相似度筛选的干扰技能，以及一个主题相似但功能相悖的对抗性“影子”技能）。仅当路由模型对所有查询均能正确选择目标技能时，描述方被判定有效。

基于该Oracle，团队将Delta调试算法改进并应用于语义子句层面：先由大模型将原始描述拆分为独立语义子句，再使用[DDMIN](https://zhida.zhihu.com/search?content_id=272523468&content_type=Article&match_order=1&q=DDMIN&zhida_source=entity)算法递归分割与测试，丢弃非必要子句，最终得到1-最小集合——每个保留子句均为正确路由不可或缺。

**2、真实环境验证**

为弥补模拟Oracle与真实运行环境的差异，第二阶段在实际代理中验证压缩后描述。测试团队会首先收集能触发原始技能的查询集合，用压缩后描述测试。若存在触发失败，则从已删除子句中贪婪恢复最能提升触发率的子句，直至所有查询均能正确触发。

**缺失描述生成**

上述两阶段流程适用于已有描述的技能。对于描述缺失或过短的技能，从主体中提取核心能力、触发条件、唯一标识，自动生成合规描述。生成的描述在通过第一阶段的模拟预言机验证后方被接受；第二阶段真实触发验证则应用于抽样子集。

### 2.2 阶段2：技能主体重构（渐进式披露）

针对主体内容冗余、引用全量加载的问题，自动将单体架构技能转换为**分层架构** ：核心内容常驻上下文，补充内容按需加载。

**1、分类驱动的内容拆分**

对技能体的段落级内容进行自动标注，将段落分为核心规则、背景、示例、模板、冗余五类；仅核心规则保留在常驻核心模块，其余内容转为独立的按需加载模块，冗余内容直接删除。

**2、分类型定制化压缩**

核心规则做语义合并与精简，示例/模板按概念去重仅保留代表性内容，背景内容做精准摘要，同时完成外部引用文件与主体的跨文件去重，为保留的引用添加触发条件、主题关键词等路由元数据。

**3、双重质量门+自校正反馈循环**

  * **Gate1 忠实性验证** ：校验压缩后内容是否完整保留了原技能的所有核心操作概念，失败则回滚对应内容类型。
  * **Gate2 任务化功能评估** ：为每个技能生成5类测试任务，通过代码执行断言+LLM评审双机制验证功能完整性；对失败任务，自动识别并将必要的非核心内容提升至常驻核心，最多2轮迭代，确保功能不退化。



## 三、核心实验结果

论文在600个技能、SkillsBench基准测试上完成了全面评估，核心结论如下：

### 显著的token缩减效果

  * **路由描述** ：平均**48%压缩率** ，已有完整描述的技能压缩率达56.5%。
  * **技能主体** ：平均**39%压缩率** ，单技能平均节省约1000token；SkillsBench基准上主体压缩率达75%；野生超大技能（>10K Token）核心压缩率可达95.8%。
  * **端到端** ：平均节省26.8%的输入Token，压缩600个技能仅需14-18美元，几百次调用即可覆盖成本。



### 功能保留与反常识的性能提升

  * **路由层** ：非过时技能实现100%路由保留。
  * **功能表现** ：86.0%的技能压缩后表现不低于原版，25.3%的技能表现反而提升，整体功能质量提升2.8%，验证了**少即是多效应** ——移除非必要内容减少了LLM的上下文干扰；SkillsBench上87个任务全部通过，零功能退化。
  * 真实压缩失败率仅4.7%，其余退化均来自技能过时、评估噪声。



## 四、总结

SkillReducer的提出对LLM代理生态的发展具有多重价值。在短期层面，该框架能够直接降低大模型应用的Token成本，提升代理的执行性能和输出质量，对于大规模部署LLM代理的企业而言，能够显著降低运营成本。研究团队计划开源完整的去膨胀流水线，作为技能作者的构建时预处理工具，帮助开发者从创作阶段就构建高效的技能。

从长期来看，这项工作揭示了技能作为新兴软件制品需要建立完整的优化生态，正如传统软件拥有编译、优化、混淆等成熟工具链一样。技能工程正在成为一个独立的研究方向，未来行业需要建立技能优化的统一标准，开发更智能的动态按需加载机制，推动整个技能市场的质量提升。

随着LLM代理在各行各业的落地应用，技能作为代理能力的核心载体，其效率和质量将直接决定大模型技术的落地成本和用户体验。SkillReducer的研究成果为解决技能膨胀问题提供了可行的技术路径，也为技能工程领域的未来发展奠定了重要基础。
