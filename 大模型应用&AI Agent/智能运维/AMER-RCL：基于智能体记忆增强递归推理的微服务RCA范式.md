
微服务架构已成为分布式系统的核心范式，但成百上千个强依赖的细粒度服务，极易因单点故障引发级联告警，快速精准的根因定位（RCL）成为保障系统可靠性的核心需求，也是AIOps领域的研究热点。

在2026年ICSE-SEIP大会上，北京大学、清华大学、阿里巴巴等机构的联合团队发布了AMER-RCL框架，该方案从SRE专家的人工诊断流程中提炼核心方法论，同时破解了现有根因定位方案“准确率不足”与“推理效率低下”两大核心痛点，在性能上全面超越业界最优方案，为LLM驱动的微服务故障诊断提供了全新的工业级范式。

# 一、行业痛点：现有根因定位方法的核心瓶颈

微服务根因定位的核心，是在复杂依赖与海量数据中区分故障根因与次生异常，锁定最小故障单元。现有两类主流方案均存在难以突破的局限：

## 1.1 传统方法的固有缺陷

- **图分析类方法**：通过依赖图、因果图建模组件关系，可解释性强，但依赖预定义的刚性图结构，无法适配动态变化的系统环境，跨场景泛化能力差。
- **深度学习类方法**：自动学习多源数据的时空与结构依赖，但黑盒特性导致推理不可解释，模型迁移能力受限，且需要大量标注数据。

## 1.2 现有LLM方法的两大核心短板

大语言模型虽解决了传统方法的泛化性难题，但现有方案仍存在致命缺陷：

- **以症状为中心的浅层推理，准确率不足**：大多采用通用的CoT、ReAct框架，仅围绕告警症状做线性分析，无法在级联故障中区分根因与次生异常，复杂场景下准确率大幅下降。
- **缺乏跨告警推理复用，推理效率极低**：对每条告警独立执行完整推理流程，而生产环境中一个故障窗口往往产生数百上千条相关告警，大量数据重叠带来严重的计算冗余，推理延迟无法满足应急需求。

# 二、核心洞察：SRE专家人工诊断的三大特征

为破解痛点，研究团队联合22位资深SRE与开发工程师，通过实证研究提炼出专家根因诊断的三大核心特征，这也是AMER-RCL的设计核心：

1. **递归性**：SRE从告警入口链路开始，沿服务调用链递归向下钻取，逐层缩小排查范围，同时对未穷尽的路径回溯，确保无遗漏。
2. **多维度拓展**：同步开展两个维度的排查——垂直拓展穿透服务、实例、Pod、宿主机等层级，排查底层资源故障；水平拓展关联同一故障窗口的所有告警，区分系统性故障与单点异常。
3. **跨模态推理**：通过链路（Trace）、指标（Metrics）、日志（Logs）三类数据交叉验证，先通过链路划定可疑范围，再通过指标、日志确认根因，排除次生异常干扰。

# 三、方案详解：AMER-RCL框架的核心设计

AMER-RCL以多智能体协同为基础，通过**递归推理RCL引擎**解决浅层推理的准确率问题，通过**Agentic Memory（智能体记忆）** 解决跨告警推理复用的效率问题。

其**核心工作流**为：输入时间窗口内的告警，先为每条告警构建因果图，再通过智能体记忆匹配历史推理结果，最终由递归推理引擎完成深度分析，输出排序后的根因候选集。

## 3.1 Agentic Memory

**因果图提取：告警数据的结构化建模**

框架为每条告警构建确定性的因果图，解决LLM难以处理海量非结构化数据的问题：

- 图节点对应系统组件，属性包含服务ID、Pod ID、指标统计、日志异常等核心信息；
- 图边对应服务间的调用与因果依赖，属性包含调用延迟、响应状态码等关键指标。

这种结构化表示既保留了故障完整上下文，又大幅压缩了无效信息，为后续匹配与推理提供标准化输入。

**记忆沉淀和复用**

智能体记忆是框架提升效率的核心，它是一个结构化、可查询的知识库，以告警因果图为键，以完整推理轨迹为值，实现知识的增量积累与高效复用。

- **记忆积累**：每完成一条告警的推理，系统会写入包含因果图指纹、向量嵌入、完整因果图、元数据与结构化推理轨迹的记忆条目。
- **记忆利用**：针对新告警，通过相似度匹配分为三类场景执行差异化策略：高相似度直接复用历史结果；部分相似度仅针对差异节点启动推理；低相似度执行完整推理并沉淀新的记忆。

该机制从根本上解决了同类告警的重复推理问题，大幅降低了计算冗余。

## 3.2 递归推理RCL引擎：深度保障的多智能体分析

递归推理引擎是框架提升准确率的核心，它以MetaAgent为总控，复刻SRE的递归诊断流程，并通过三阶段框架避免LLM推理提前终止。

**专用智能体分工**

框架设计了各司其职的智能体，精准对应诊断流程的不同动作：

- TraceAgent、LogAgent、MetricAgent分别负责链路、日志、指标数据的精准筛选与异常提取；
- Consolidator负责整合多源结论，为候选根因生成结构化证据链；
- MetaAgent作为总控，协调各智能体、生成推理指令、驱动递归与回溯流程。

**核心递归推理算法**

算法核心逻辑为：从告警入口Span启动分析，通过链路数据判断节点是否可疑；对可疑节点调用日志、指标智能体做跨模态验证，确认根因则加入候选集；未确认则递归遍历所有可疑子Span，全路径分析完成后，整合生成最终的根因排序列表。通过递归+回溯机制，彻底解决了线性推理的浅层分析问题。

**深度保障三阶段框架**

针对LLM易提前终止推理的问题，框架设计了三阶段强制保障机制：

- **初始推理阶段**：仅通过链路数据完成全链路递归遍历，快速划定可疑范围，避免推理过早收敛；
- **批判性反思阶段**：对所有可疑节点强制做跨模态深度验证，继续递归下钻未穷尽路径；
- **最终评审阶段**：整合全量推理轨迹，统一做证据聚合与根因排序，消除LLM的近期偏见。

# 四、实验验证：全面超越SOTA的性能表现

研究团队在AIOPS 2022、Train-Ticket、FAMOS-Mall三个业界基准数据集上，对AMER-RCL开展了全面验证，结果如下：

**根因定位准确率**

![alt text](image-1.png)

AMER-RCL在三个数据集上的MRR分别达到**76.30**%、**81.93**%、**75.39**%，平均超出次优SOTA方法（mABC）15.75%，Top1/Top3/Top5召回率均全面领先非LLM与LLM类基线方案。

**推理效率**

![alt text](image-2.png)

批量告警场景下，AMER-RCL平均推理速度比RCAgent快**3.53**倍，比CoT方案快**14.61**倍，比mABC快**31.05**倍，在同质化告警场景下加速效果更为显著。

**适配性与消融实验**

![alt text](image-3.png)

- 框架可适配不同能力的LLM骨干，Claude-3.5 Sonnet表现最优，平均超出次优模型18.14%；
- 消融实验验证了各组件的有效性：批判性反思环节带来13.21%的MRR提升，最终评审贡献2.96%，智能体记忆模块带来2.61%的稳定提升。

# 五、总结

AMER-RCL一方面用递归推理引擎突破了浅层推理的准确率瓶颈，另一方面引入智能体记忆模块，有效解决了跨告警场景下的冗余推理问题。其它从真实工业实践出发，用LLM与多智能体技术实现了专家能力的沉淀与规模化复用，为云原生时代的系统可靠性保障开辟了全新的可落地路径。
