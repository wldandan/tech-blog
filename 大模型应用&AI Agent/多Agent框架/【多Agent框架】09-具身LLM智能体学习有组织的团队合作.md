目录

本篇文章将会对论文《Embodied LLM Agents Learn to Cooperate in Organized Teams》进行分享。在这篇文章中，**主要围绕当前主要的LLM Agents在合作完成任务过程中遇到的幻觉放大、过于发散的问题，提出了一个基于组织结构的LLM Agents框架。**

## **背景信息**

当下大部分基于LLM的多Agents之间协作通常采用启发式设计的预定义模式，实践证明这种模式使得LLM Agents经常发送冗余和重复的信息，并中断其他Agents行为，导致混乱。如下图展示了在一个由三个GPT-4 Agents组成的团队中，没有指定领导者，两个智能体进行了不必要的交流，做出了无序决策。

本文主要围绕这个问题，借鉴从认知和经济角度对人类协作的研究，主要研究如下两个问题：

  1. 组织结构在多LLM Agents系统扮演什么角色？
  2. 如何优化组织结构来支持多Agents的高校协作？



为了解决上述问题，作者们基于[AutoGen](https://zhida.zhihu.com/search?content_id=252248501&content_type=Article&match_order=1&q=AutoGen&zhida_source=entity)开发了一款能够在多Agents系统快速使用各种组织结构的框架，其主要研究如下：

  1. 以典型的分层结构入手，研究得出领导者可以在多Agents系统大大提高效率。
  2. 发现在适当的组织下，LLM Agents表现出模仿人类的各种合作行为。例如，Agents可以提供建设性的建议并寻求他人的帮助；他们还可以为层次结构执行适当的交互，例如报告任务进度。
  3. 本文还测试了人-Agents协作，并观察到与AI Agents相比，人类领导者在协调一组Agents方面要好得多。
  4. 本框架还实现了批判反思能力，采用双重LLM架构，通过执行结果生成改进组织结构的prompt，在不断迭代过程中多Agents自发形成了新颖有效的团队结构。



## 技术方案

### 架构和多LLM-Agents通信

整体框架如下：

上面左图可以看到该框架增加了四个模块：

  * **配置器（Configurator）：** 配置Agents的信息。
  * **感知模块（ Perception Module）** ：将环境观察结果转换为文本。
  * **记忆模块（Memory Module）** ：存储和检索历史信息。
  * **执行模块（ Execution Module）** ：任务或者动作执行。



右图展示了将一次执行分为沟通和行动两个阶段，在沟通阶段Agents通过广播或者定向发送消息。本框架具有如下能力：

  * 使用两个不同的LLM（Actor和Communicator）完成通信决策和行动决策。
  * 利用prompts为Agents团队强加了一个组织结构，比如给Actor和Communicator的prompts增加了文本描述。
  * LLM Agents在执行任务过程中不断在沟通阶段和行动阶段切换，沟通阶段支持丰富的团队结构和灵活的沟通模式。
  * 在通信过程中，Agent轮流进行通信。它既可以发送广播消息、也可以为消息选择一个或者多个收件人并向其发送不同的消息，还可以保持静默。Agent保留自己的通信历史记录，并可以响应来自以前通信的消息。



### 批判反思优化组织结构

如下图所示，本框架用两个LLM实现批判反思，从而优化组织结构：

  * **LLM批判员：受到强化学习的[Actor-Critic方法](https://zhida.zhihu.com/search?content_id=252248501&content_type=Article&match_order=1&q=Actor-Critic%E6%96%B9%E6%B3%95&zhida_source=entity)的启发，框架引入了LLM批判员，根据口头反馈来评估团队的表现。**批判员将一个迭代的对话和行动历史作为输入。然后分析输入和原因，提取和总结认为影响效果的关键步骤。此外，批判员还对Agents的行为和他们的领导地位进行了文本评价。
  * **LLM协调员** ：**LLM协调员将LLM批判员的输出以及来自环境的成本度量（任务完成时间和沟通成本）作为输入。** 它根据这些数据进行反思，并根据对过去的事件和最初的例子的分析产生思考。通过思考组织结构prompts及其效果，协调员会为下一次迭代提出新的和不同的组织结构prompts。



## 主要成果

### 指定领导者增强效果

通过对照实验研究了组织结构和领导者对多Agents系统的影响，实验结果如下图：

  
上图列出了无领导者、固定领导者、选举领导者和人类作为领导者四种情况下任务完成时间和通信花费对比。  
本文对20组实验结果做了双样本t检验，数据表明效果在统计学意义下显著提高了9.76%。相比无组织结构，指定领导者对沟通成本的影响很小。与指定领导者相比，选举产生领导者提高了团队的执行效果（t(38) = 1.84, p < .05），但是同样会大大增加通信成本。人类领导的团队完成任务沟通效率更高。

### **领导力和开放沟通比较重要**

在混合使用GPT-4和GPT-3.5-turbo Agents的团队中，任命GPT-4为领导者比GPT-3.5-turbo为领导者的团队效率提高得更多。本文分别在3个Agents和5个Agents的团队上进行了这个实验，这两种情况下，GPT-4作为领导者时，任务完成时间和沟通成本都会减少。这一发现意味着这些LLM之间的领导水平不同。   
通过实验还观察到，鼓励对领导Agents的建设性反馈有助于提高效果。我们通过添加一个额外的prompt来促进LLM Agents之间的开放沟通：“如果领导的指示不正确，你可以纠正领导”。有趣的是，当团队由3xGPT-4成员组成时，这一修改提高了团队的整体效率，并减少了任务完成的时间（t(38) = 0.87, p = 0.14）。相反，当GPT-3.5-turbo Agents试图纠正领导者，同样的修改会降低团队效率，通信成本增加。

### 合作行为的产生

本文为了研究LLM Agents系统中合作行为的产生，按照如下步骤进行了实验：

  1. 定义了三大类合作模式：（i）信息共享：Agents通过主动提供新信息或通过被请求来影响他人。向领导汇报、分享新的观察结果以及回答问题都属于这 一类。（二）领导与协助：Agents，特别是领导者（如果有的话）可以通过改变计划来影响他人。这些行为包括任务分配、纠正、求助等。（iii）请求指导：Agents主动请求新的信息或计划，用于自己的决策。 
  2. 开发了一个独立的基于提示的GPT-4分类器来分析每一段对话。在20个人类标记的对话样本（60个标签）上测试时，分类器的准确率为91.67%。 
  3. 使用分类器来标记由Agents生成的消息，并报告具有合作行为的消息的百分比 （一个消息可能有多个标签）。



最终的实验结果如下图所示：

结果表明，即使在一个组织混乱的团队中，LLM的Agents也喜欢告诉别人该做什么。领导和帮助大约占所有行为的 50%以上（图（a）)。然而，除了告诉其他人该做什么，杂乱无章的团队中的Agents并没有表现出太多的合作行为，例如，他们会在不到10%的对话中请求指导。 相比之下，当团队有一个层级组织时，牵头的LLM Agent将假定一个主导角色并向其他人发出命令（占他们的通信量的60%以上），与无组织的情况相比，其他成员倾向于跟随并发出较少的命令（图(b, c）)。在这样的团队中，Agents倾向于分享和询问更多的信息。

上图表明，系统初始化时有一个基本的组织提示，即“Agent_1作为领导协调任务”。随着反思过程的推进，协调者会生成一系列不断演变的组织提示，选取诸如“分层”和“ 动态”这样的关键词，这些关键词暗示了更复杂的团队结构。 通过实验比较了团队在批评-反思步骤前后的表现。上图(b)说明了团队的效率，我们观察到，对于3×GPT-3.5- turbo，新的组织结构提高了团队完成任务的效率(t(38)=1.73,p<0.05)，但通信成本略有增加。而对于 3×GPT-4 和 1×4+2×GPT-3.5-turbo ，通信开销随着任务效率的提高而降低 (t(38)=1.56,p=0.06,3×GPT-4,t(38)=0.32)。1×4+2×GPT-3.5-turbo )。

## 总结与展望

本文开发了一款**新的LLM-Agent架构来促进Agents之间通信和组织具身Agents团队加强合作。本框架利用基于LLM的批评反思机制来生成更有效的组织结构prompt** 。对各种组织结构的广泛实验表明，具有指定/选举领导者的分层组织团队具有卓越的团队效率，可以通过批评-反思进一步提高团队效率。

目前的工作是在单一的环境中进行的，缺乏人的评估。未来的工作将扩展到更广泛的环境，允许人类评估。由于[VirtualHome](https://zhida.zhihu.com/search?content_id=252248501&content_type=Article&match_order=1&q=VirtualHome&zhida_source=entity)无法容纳数百个智能体，未来的工作也可以在其他环境中探索更大的组织。
