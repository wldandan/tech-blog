## 背景介绍

尽管开源大型语言模型（如 [LLaMA](https://zhida.zhihu.com/search?content_id=255494914&content_type=Article&match_order=1&q=LLaMA&zhida_source=entity)）取得了进步，但它们在**工具使用能力** （即使用外部工具（API）来实现人类指令）**方面仍然受到很大限制** 。究其原因，目前的指令调整主要集中在基本语言任务上，而忽略了工具使用领域。这与最先进的闭源 LLM（如 [ChatGPT](https://zhida.zhihu.com/search?content_id=255494914&content_type=Article&match_order=1&q=ChatGPT&zhida_source=entity)）出色的工具使用能力形成了鲜明对比。但它们都是闭源的，其内部机制并不透明。这限制了人工智能技术的普及以及创新发展。为了弥补这一差距，当务之急是让开源 LLM 能够熟练掌握各种 API。我们引入了 **[ToolLLM](https://zhida.zhihu.com/search?content_id=255494914&content_type=Article&match_order=1&q=ToolLLM&zhida_source=entity) ，这是一个包含数据构建、模型训练和评估的通用工具使用框架**。

## 技术方案

**ToolBench** 是一个使用 ChatGPT 自动构建的工具使用指令调整数据集。具体来说，构建过程可分为三个阶段：

  1. API 收集：从 RapidAPI Hub 收集了 16,464 个真实世界的 RESTful API，涵盖 49 个类别；
  2. 指令生成：促使 ChatGPT 生成涉及这些 API 的各种指令，涵盖单工具和多工具场景；
  3. 解决方案路径注释：使用 ChatGPT 为每个指令搜索有效的解决方案路径（API 调用链）。



为了增强 LLM 的推理能力，作者开发了一种新的基于深度优先搜索的决策树算法。它使 LLM 能够评估多个推理轨迹并扩展搜索空间。此外，为了评估 LLM 的工具使用能力，作者开发了一种自动评估工具： **[ToolEval](https://zhida.zhihu.com/search?content_id=255494914&content_type=Article&match_order=1&q=ToolEval&zhida_source=entity)**. 在 ToolBench 的基础上，作者对 LLaMA 进行了微调，得到了 **LLM ToolLLaMA** 。

如上图，构建 ToolBench 的三个阶段，以及如何训练 [API retriever](https://zhida.zhihu.com/search?content_id=255494914&content_type=Article&match_order=1&q=API+retriever&zhida_source=entity) 和 ToolLLaMA。在指令推理过程中，API retriever 会向 ToolLLaMA 推荐相关的 API，而 ToolLLaMA 则会执行多轮 API 调用，从而得出最终答案。整个推理过程由 ToolEval 进行评估。

## 实验

### 初步实验

作者开发了基于 ChatGPT 的高效评估工具 ToolEval，其中包含两个评估指标：

  1. 通过率：计算在有限预算内成功完成指令的比例。该指标衡量的是 LLM 指令的可执行性；
  2. 胜出率：作者向 ChatGPT 评估器提供一条指令和两条求解路径，并获得其偏好。同时又为这两项指标预先定义了一套用作 ChatGPT 评估器提示的标准。通过进行多次评估后计算平均结果。



测试结果表明 ToolEval 与人类注释者的一致性很高，通过率为 87.1%，胜出率为 80.3%。**这说明 ToolEval 可以在很大程度上反映和代表人类的评价。**

API 检索器旨在检索与指令相关的 API。作者的应用程序接口检索器在所有情况下的表现都优于基线，这表明它在具有海量应用程序接口的真实世界场景中是可行的。

### 主体实验

本文在三个层面上评估 ToolLLaMA 的泛化能力：

  1. Inst.：训练数据中同一组工具的未见指令；
  2. Tool：属于训练数据中工具的相同（可见）类别的未见工具；
  3. Cat.：属于训练数据中工具的不同（未见）类别的未见工具。



作者在三种情况下进行了实验：单工具指令、类别内多工具指令和集合内多工具指令。对于每个测试指令，作者都会向每个模型输入地面实况（oracle）API。这就模拟了用户指定自己喜欢的 API 集的场景。

实验通过将每个模型与 [ChatGPT-ReACT](https://zhida.zhihu.com/search?content_id=255494914&content_type=Article&match_order=1&q=ChatGPT-ReACT&zhida_source=entity) 进行比较来计算胜率。胜率高于 50%，表示该模型的表现优于 ChatGPT-ReACT。

结果表明，**ToolBench 可以充分激发 LLM 的工具使用能力，使他们能够熟练掌握各种指令的未知API** 。

## 总结

本文介绍了如何在 LLM 中激发工具使用能力。作者首先介绍了一个指令调整数据集 ToolBench，该数据集涵盖了16000多个真实世界 API 和各种实际用例场景，包括单工具和多工具任务。ToolBench 的构建完全使用 ChatGPT，只需极少的人工监督。为了更高效地评估，作者设计了自动评估工具 ToolEval。通过在 ToolBench 上对 LLaMA 进行微调，所得到的 ToolLLaMA 模型与 ChatGPT 的性能相匹配，并对未见过的 API 展示出卓越的泛化能力。此外，作者还开发了神经 API 检索器，为每条指令推荐相关的 API。该检索器可与 ToolLLaMA 集成，成为更加自动化的工具使用管道。总之，这项工作为未来在 LLM 的指令调整和工具使用交叉领域的研究铺平了道路。
